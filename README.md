  以下是对第前期线上仿真实现的代码一些说明
一、动力学模型（packaged_drone_dynamics_model)
设置了一个类（表示单个无人机），类的成员变量为无人机的位置，速度和模拟的时间步长，这个类决定了无人机当前的状态；类的方法包括一个类的初始化，更新类成员变量（模拟无人机的运动（使用欧拉法，先计算速度再计算位置），和得到当前状态（以一个numpy的数组表示，方便后续计算）
二、控制器的编写(controller)
设置一个类（表示控制器（主要用于计算）），一架无人机对应一个控制器，类的成员即PID的参数（k，p，i以及最大积分量），类的方法为初始化PID控制器（在模拟中无空气阻力存在，故无稳态误差，所以i为0），计算加速度，为了减小从机误差加入了前馈控制，即将主机加速度当作从机目标加速度一部分参与加速度计算，减小反应时间从而减小误差。

四、PID最佳参数的获取（pid_tuner)
通过仿真评估pid参数,使位置误差最小化并返回最优参数,在twiddle中逐维调整参数.
对每个参数 i：尝试增加：p[i] += dp[i]，
若误差变好（err < best_err），接受改动并把 dp[i] *= 1.1（扩大步长）。
否则，尝试减小：p[i] -= 2*dp[i]（从原点向相反方向移动），若变好则接受并 dp[i] *= 1.1。
若两侧都不行，恢复原值 p[i] += dp[i] 并缩小步长 dp[i] *= 0.9。
终止条件: 所有 dp 之和小于容差 tol（默认为 0.2）。返回最终 p 和 best_err。
对发散或非法参数返回 +inf，会把这些区域排除在可接受解之外，从而保证稳定性偏好
通过返回 +inf 或累积远离目标的平方误差来惩罚不稳定或高超调解；可以扩展为加入控制量大小的惩罚项（比如加速度或控制能量）以得到更平衡的控制器。
1、初始化	
在编队控制中先设置时间步长和模拟时间（步长决定精度），然后初始化了一个无人机作为主机，然后设置各个从机和主机的偏差，将偏差量与主机位置结合作为从机位置，依此初始化从机。为主机设置航点（在我写的文件里面是一个圆）和判断主机是否到达的数据
2、控制环节
在控制中主机不需要前馈调节，不传入加速度（默认为None），当判断主机到达航点时，航点索引加一，主机控制器和速度状态更新；对于从机，在目标上以主机相关参数为目标，在控制器的使用时传入主机当前加速度作为目标加速度
四、图像绘制
使用plotly库以得到可动三维立体图像，在程序中建立名为history的列表以存储运动位置数据从而实现绘图，对于从机，同样建立了列表存储误差，得到各时刻位置误差图
五、PID
PID是常见的控制算法，利用与目标的偏差实现控制量输出，PID包括比例控制（P），积分控制（I）和微分控制（D），P决定相应速度，I消除稳态误差，D抑制超调
P的参数一般直接为误差，I的参数为误差的积分（在代码中使用x*dt表示），D的参数为误差的微分（在代码中使用速度表示）由于模拟中为理想状态，并没有进行滤波和微分量，PID计算得到的结果作为控制量输出。
在代码中加入了前馈调节，PID属于后馈调节，具有滞后性，而前馈调节的加入提高了算法的响应速度，显著减小了误差。
