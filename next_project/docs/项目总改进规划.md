# 四旋翼集群编队仿真系统总改进规划

更新日期：2026-05-03

本文档汇总当前代码、技术文档、GNN 分层双模式方案、路径规划改进方案和本地参考文献评估，形成后续改进的统一执行路线。规划目标不是一次性重写系统，而是在已验证的零碰撞与低误差基线上，逐步提升在线规划质量、动态障碍处理能力、轨迹连续性和容错能力。

文档分工：`docs/技术文档.md` 只保留已实现系统的技术结构、数学模型、模块接口和测试入口；本文档记录改进思路、实施顺序、参考文献可借鉴点、风险取舍和下一步规划。

## 2. 改进原则

所有改进必须遵守以下约束：

1. 不绕过现有五层安全链路：规划、SDF 校核、线段采样、APF、控制器跟踪必须保持分层。
2. 不把论文公式直接替换工程模块，必须先明确输入输出量纲、注入点和回退路径。
3. 不在简单场景默认启用高风险功能，例如故障检测、强化学习、强耦合编队势场。
4. 每个改动必须有最小可运行测试，先测模块，再测场景。
5. Python 主线稳定后，再决定是否同步 `cpp/`。

## 3. P0 改进：在线规划可靠性

P0 是下一轮最优先改进，目标是减少在线场景中的局部规划失败、路径跳变和无意义重规划。

### 3.1 GNN 可见图路径质量闭环

当前已修复 GNN 路径提取跳过可见边的问题，并已补齐质量闭环。

| 项目 | 内容 |
|---|---|
| 目标 | 保证 GNN Danger 路径可见、可达、长度不过度劣化 |
| 涉及文件 | `core/planning/gnn_planner.py`、`visibility_graph.py`、`tests/test_gnn_planner.py` |
| 改动 | 已增加路径段可见性校验、路径长度与 A* 局部路径对比、GNN 失败时显式返回 fallback 原因 |
| 验收 | 已完成 `tests/test_replanner_danger_quality.py`；GNN 路径不可见或长度劣化时回退局部 A* |
| 风险 | 可见图过密导致耗时上升 |
| 回退 | `danger_mode_enabled=False` 或 Danger 模式直接使用局部 A* |

### 3.2 受限视野辅助目标

Voronoi/NMPC 文献强调受限感知条件下不应直接追远端目标，而应在可见范围内选择辅助目标。

| 项目 | 内容 |
|---|---|
| 目标 | 在线模式在有限传感器视野中选择更稳定的局部 subgoal |
| 涉及文件 | `core/planning/replanner.py` |
| 改动 | 将 `_compute_subgoal()` 从直线 horizon 截断升级为 SDF 可见、风险低、靠近全局路径的辅助目标选择；已加入候选点评分、线段安全检查和局部自由前沿补充 |
| 验收 | 已完成单测 `tests/test_replanner_subgoal.py`；相关障碍规划回归 `5 passed, 18 deselected` |
| 风险 | 辅助目标过保守导致任务推进变慢 |
| 回退 | 保留原直线 horizon subgoal 作为 fallback |

### 3.3 动态障碍语义层

MIIM 文献的核心价值是静态层与传感器语义层分离。当前已完成静态层保护，并补齐动态障碍语义层的生命周期管理。

| 项目 | 内容 |
|---|---|
| 目标 | 将传感器命中的动态障碍与静态障碍明确分层 |
| 涉及文件 | `core/planning/replanner.py`、`core/obstacles.py` |
| 改动 | 已为 `_sensor_occupied` 增加 TTL、清空确认和 `_decay_sensor_obstacles()` 衰减策略 |
| 验收 | 已完成 `tests/test_replanner_semantics.py`；动态障碍消失后可恢复通行，静态障碍不被清空 |
| 风险 | TTL 过短漏障碍，过长堵通道 |
| 回退 | 关闭 TTL，保持当前 `_sensor_occupied` 行为 |

### 3.4 在线任务航点语义闭环

`school_corridor_online`、`company_cubicles_online` 等场景中曾出现执行轨迹接近直线、未严格经过任务航点的问题。根因是在线重规划得到的 `new_path` 被用于替换当前执行航点序列，导致任务航点进度与局部重规划路径进度混在一起。

| 项目 | 内容 |
|---|---|
| 目标 | 在线重规划只生成局部执行路径，不改写原始任务航点语义 |
| 涉及文件 | `simulations/obstacle_scenario.py`、`simulations/visualization.py`、`tests/test_obstacle_scenario.py` |
| 改动 | 将 `current_wp_idx` 与 `local_path_idx` 分离；`new_path` 只更新 `local_path`；完成判断改为 leader 到当前任务航点的距离 |
| 可视化 | 分层输出 `task_waypoints`、`replanned_waypoints`、规划路径和执行路径，便于区分原始任务点与局部重规划点 |
| 验收 | `test_online_mode_preserves_task_waypoint_semantics` 通过；在线完成计数不再随局部路径点数量变化 |
| 风险 | 局部路径过短、APF 过强或任务航点半径过小仍可能导致接近但不进入判定半径 |
| 回退 | 关闭在线重规划，或使用离线 `planned_path` 执行 |

### 3.5 终端保持与末端抖动抑制

离线模式在即将到达终点时曾出现快速抖动。根因通常不是全局路径错误，而是最终航点附近目标切换、控制器积分残留和速度残差叠加。

| 项目 | 内容 |
|---|---|
| 目标 | 进入最终航点半径后稳定结束，避免终点附近高频抖动 |
| 涉及文件 | `simulations/formation_simulation.py`、`simulations/obstacle_scenario.py`、`tests/test_obstacle_scenario.py` |
| 改动 | 进入最终航点半径后记录 `terminal_hold_pose`；重置 `leader_ctrl`；在保持窗口内固定目标点 |
| 验收 | `test_terminal_hold_reduces_end_jitter` 通过；末端轨迹尾段漂移受限 |
| 风险 | 保持点是进入半径时的位置，不一定与数学终点完全重合 |
| 回退 | 将终端保持窗口设为 0 或恢复原终点跟踪逻辑 |

## 4. P1 改进：轨迹质量与动力学可行性

P1 目标是让路径从“能走”变成“更平滑、更可控、更接近动力学可行”。

### 4.1 FIRI 多段走廊轨迹优化

当前 FIRI 主要做分段精修，尚未形成完整多段时间化轨迹。

| 项目 | 内容 |
|---|---|
| 目标 | 在凸走廊内生成连续、低曲率、满足速度/加速度约束的轨迹 |
| 涉及文件 | `core/planning/firi.py`、新增 `trajectory_optimizer.py` |
| 改动 | 将各段 `FIRICorridor` 串联，求解最小 snap/jerk 或二次规划轨迹 |
| 验收 | 最大曲率下降，leader 跟踪 RMS 下降，碰撞为 0 |
| 风险 | 优化器失败或耗时过高 |
| 回退 | 使用当前 FIRI 点投影和弧长重采样 |

### 4.2 APF 增强分档落地

改进 APF 文献中的旋转势场、目标距离衰减、通信约束、编队整体势场不应全部默认开启。

| 项目 | 内容 |
|---|---|
| 目标 | 通过 profile 控制 APF 增强项，避免多参数耦合 |
| 涉及文件 | `core/planning/dual_mode.py`、`simulations/obstacle_scenario.py`、`config.py` |
| 改动 | 已接入 `off / conservative / aggressive` 三档；默认只启用低风险项 |
| 验收 | 已由在线场景和障碍规划回归覆盖；aggressive 档仅用于对比实验 |
| 风险 | 编队重心势场掩盖从机贴障碍风险 |
| 回退 | `apf_paper1_profile="off"` |

### 4.3 跟踪层 MPC 化评估

Cooperative UAV 论文采用上层规划 + MPC 跟踪。当前项目使用 PID/Backstepping/SMC，已经稳定，但轨迹突变时仍可能产生瞬态误差。

| 项目 | 内容 |
|---|---|
| 目标 | 评估是否需要轻量 MPC 轨迹跟踪层 |
| 涉及文件 | 新增 `core/planning/mpc_tracker.py` 或控制器扩展 |
| 改动 | 先做离线轨迹后处理，不直接替换控制器 |
| 验收 | 跟踪 RMS 下降，控制输入无明显饱和增加 |
| 风险 | 引入求解器依赖和实时性问题 |
| 回退 | 继续使用 Backstepping + SMC |

## 5. P2 改进：容错与长期能力

P2 属于增强项，不应影响当前已验证的在线避障主线。

### 5.1 容错拓扑端到端场景

当前已有故障注入、故障检测、拓扑图基础和端到端在线场景。

| 项目 | 内容 |
|---|---|
| 目标 | 故障后自动检测、重构队形并继续完成任务 |
| 涉及文件 | `core/drone.py`、`core/fault_detector.py`、`core/topology.py`、`config.py` |
| 改动 | 已新增 `fault_tolerance_online` 场景，注入单旋翼衰减或单机退出 |
| 验收 | 已完成 `tests/test_fault_tolerance_online.py`，故障注入、检测和重构日志可追踪 |
| 风险 | 风扰或跟踪误差误判为故障 |
| 回退 | 简单场景默认 `fault_detection_enabled=False` |

### 5.2 动态多障碍分区策略

Voronoi/NMPC 文献可用于解决多障碍最近目标频繁切换导致的振荡。

| 项目 | 内容 |
|---|---|
| 目标 | 多障碍环境中选择稳定的避障侧和局部通道 |
| 涉及文件 | `replanner.py`、可能新增 `voronoi_region.py` |
| 改动 | 基于障碍物中心构建局部 Voronoi 分区，稳定选择下一 cell |
| 验收 | 障碍密集场景重规划抖动减少，路径长度不显著增加 |
| 风险 | 3D 障碍分区复杂，收益不确定 |
| 回退 | 只在 2D 水平投影上启用 |

### 5.3 MADDPG 仅保留研究接口

当前 MADDPG PDF 文件为空，且引入强化学习会显著增加依赖和训练成本。

| 项目 | 内容 |

| 涉及文件 | 暂无 |
| 改动 | 等完整 PDF 和训练需求明确后再评估 |

## 8. 不建议短期实施的事项

| 事项 | 原因 |
|---|---|
| 完整 MADDPG | PDF 当前为空，训练数据和依赖未定义，短期不可复现 |
| 全面替换控制器为 MPC | 当前控制器已修复主要稳定性问题，直接替换风险大 |
| 默认启用故障检测 | 简单场景可能误触发，影响已有回归 |
| 强制使用 GNN 替代 A* | GNN 当前是危险区局部规划增强，不是全局最优规划替代 |
| 一次合入全部论文功能 | 难以定位回归来源，与既有反效果复盘结论冲突 |

## 11. 下一步建议

P0 可靠性补强已基本落地，下一步建议转到 P1-1“FIRI 多段走廊轨迹优化”。重点不是继续增加避障开关，而是把当前分段走廊串联成带时间参数的连续轨迹，约束速度、加速度和曲率，从而降低在线路径切换后的跟踪瞬态误差。若优先研究密集多障碍场景，则可并行推进 P2-2 的局部 Voronoi 分区原型。

补充可靠性项：本轮已修正 `compute_esdf()` 的 EDT 输入语义，确保 ESDF 表示自由体素到最近占据体素的距离，并用 `tests/test_esdf_correct.py` 覆盖该行为。后续涉及软代价、A* 风险权重或轨迹优化代价时，应继续以该测试作为基础守门。
