新增并完成的中期强化项：

- `simulations/obstacle_scenario.py`：在线循环中加入前瞻窗口诊断、急转弯识别、RRT escape 候选路径和 `formation_adaptation_events`。
- `experiments/ablation.py`：增加 `formation_aware_lookahead_adaptive` 变体，可与 `leader_only_planner` 和 `formation_aware_adaptive` 对比。
- `experiments/metrics_extractor.py`：提取 `lookahead_reference_blocked_count`、`rrt_escape_attempt_count`、`rrt_escape_accepted_count`、`rrt_escape_failed_count`。

关键摘录：

| 场景 | 变体 | 队形调整事件 | 前瞻阻断 | RRT attempt | RRT accepted | 硬碰撞 |
|---|---|---:|---:|---:|---:|---:|
| `rrt_dual_channel_online` | `formation_aware_lookahead_adaptive` | 8 | 2 | 2 | 2 | 0 |
| `formation_maze_stress_online` | `formation_aware_lookahead_adaptive` | 11 | 3 | 3 | 3 | 0 |

该结果支撑一个明确结论：在狭长通道和大角度转弯场景中，单纯 leader-only 路径不足以解释从机安全裕度；引入编队感知 clearance、提前队形调整和 RRT 前瞻 escape 后，可以形成可记录、可消融、可报告的安全调控链路。

C++ 本轮已对齐的内容：

仍需保留的边界：

- C++ 侧是可运行子集，不宣称完整复现 Python 的 RRT* / Informed RRT* 能力。
- C++ replay 中 follower 仍是展示/回放代理，不等价于完整多机闭环动力学 benchmark。
- Python 的 `minimum_jerk` / `min_snap_proxy` 是工程原型，不宣称完整 GCOPTER/MINCO。
