# next_project 使用说明

本文面向运行和演示本项目的人，覆盖 Python 主线、C++ 子项目，以及新增的动态地图在线重规划回放系统。

## 0. 先进入项目根目录

本文所有相对路径命令都默认在 `next_project/` 目录下执行。当前工程外层还有一层压缩包同名目录，因此不要停在：

```powershell
G:\年度项目 2.zip
```

应先进入真正项目根目录：

```powershell
cd "G:\年度项目 2.zip\年度项目.zip\年度项目\next_project"
```

进入后应能看到：

```text
config.py
cpp/
docs/
maps/
simulations/
web/
```

如果你使用 conda 环境，例如 `(annual_project)`，请在该环境中安装 Web 后端依赖：

```powershell
python -m pip install -r web/requirements.txt
```

## 2. Python 主线运行

在 `next_project/` 目录下执行：

```powershell
python main.py
```

预设配置集中在：

```text
config.py
```

常用预设包括：

| 预设 | 用途 |
|------|------|
| `basic` | 基础编队验证 |
| `obstacle` | 简单障碍物避障 |
| `warehouse` | 工业仓库复杂场景 |
| `warehouse_online` | 仓库在线重规划 |
| `warehouse_danger` | 仓库 Danger/GNN 双模式 |
| `fault_tolerance` | 故障注入与容错拓扑重构 |

修改默认场景时，在 `config.py` 或调用 `get_config("preset_name")` 的入口中切换。

## 4. 动态地图在线重规划回放系统

该系统用于验证：

- D* Lite 增量重规划
- Grid A* 全局重规划基线
- 动态障碍物 ADD / REMOVE 事件
- replay JSON 回放与指标面板

相关文件：

| 文件 | 作用 |
|------|------|
| `cpp/include/dynamic_scenario.hpp` | 动态场景执行器 |
| `cpp/src/dynamic_main.cpp` | C++ CLI 入口 |
| `web/server.py` | FastAPI 后端 |
| `web/dynamic_replay.html` | 单文件前端 |
| `.claude/superpowers/next_project_docs_superpowers/specs/2026-05-04-dynamic-replay-design.md` | 设计规格 |

### 4.1 启动 Web

先构建 C++：

```powershell
cmake --build cpp/build --target sim_dynamic_replay -j 4
```

再启动后端：

```powershell
cd web
python -m uvicorn server:app --host 127.0.0.1 --port 8765
```

浏览器打开：

```text
http://127.0.0.1:8765
```

页面可完成：

- 选择预设与地图
- 调整栅格分辨率、安全裕度、仿真时长等参数
- 编辑动态障碍物事件 JSON
- 运行仿真
- 查看 2D 地图、D* Lite / A* 路径、耗时曲线和汇总指标
- 仿真结束后，在 2D 地图上方通过 `2D地图` / `3D视图` 按钮切换视图；3D 视图使用 Plotly 风格渲染路径、障碍物和编队位置
- 仿真结束后可拖动进度条查看任意帧：2D/3D 视图会显示当前帧障碍物、Leader 已走轨迹、Followers 位置和当帧规划路径
- 动态事件支持 `add`、`remove`、`move`；`move` 事件通过 `target_id` 指定已有障碍物，并在 `obstacle` 中给出移动后的类型、位置和尺寸

### 4.2 命令行运行动态回放

输入 JSON 示例：

```json
{
  "preset": "warehouse_online",
  "map_file": "sample_warehouse",
  "repeat_count": 1,
  "compare_planners": ["dstar_lite", "astar"],
  "base_config": {
    "max_sim_time": 10.0,
    "planner_resolution": 0.8,
    "safety_margin": 0.0,
    "planner_replan_interval": 0.5,
    "leader_max_vel": 2.0,
    "wp_radius_final": 0.4
  },
  "events": [
    {
      "t": 5.0,
      "action": "add",
      "obstacle": {
        "type": "sphere",
        "center_or_min": [15, 10, 3],
        "size_or_max": [1.5, 0, 0]
      }
    },
    {
      "t": 8.0,
      "action": "remove",
      "target_id": "obs_3"
    }
  ]
}
```

运行：

```powershell
cpp/build/sim_dynamic_replay.exe input.json -o replay_output.json
```

输出为 replay JSON 数组。每个元素包含：

- `metadata`
- `static_map`
- `dynamic_events`
- `frames`
- `summary`

### 4.3 动态事件格式

添加障碍物：

```json
{
  "t": 5.0,
  "action": "add",
  "obstacle": {
    "type": "sphere",
    "center_or_min": [15, 10, 3],
    "size_or_max": [1.5, 0, 0]
  }
}
```

AABB：

```json
{
  "t": 6.0,
  "action": "add",
  "obstacle": {
    "type": "aabb",
    "center_or_min": [20, 5, 0],
    "size_or_max": [22, 8, 4]
  }
}
```

Cylinder：

```json
{
  "t": 7.0,
  "action": "add",
  "obstacle": {
    "type": "cylinder",
    "center_or_min": [10.5, 2, 0],
    "size_or_max": [0.3, 0, 6]
  }
}
```

删除障碍物：

```json
{
  "t": 8.0,
  "action": "remove",
  "target_id": "obs_3"
}
```

ID 规则：

- 静态地图加载后自动生成 `obs_0`, `obs_1`, `obs_2` ...
- 动态新增障碍物自动生成 `dyn_0`, `dyn_1` ...
- REMOVE 事件使用稳定字符串 ID，不使用数组下标

## 6. 验收指标

动态回放系统建议关注：

| 指标 | 目标 |
|------|------|
| 碰撞次数 | `0` |
| 终点距离 | `< wp_radius_final` |
| 路径长度差 | D* Lite 与 A* 差异不超过 `5%` |
| 平均耗时比 | `A* / D* Lite >= 5x` |
| P95 耗时比 | `A* / D* Lite >= 5x` |

输出位置：

- CLI 模式：由 `-o replay_output.json` 指定
- Web 模式：后端返回 JSON，前端直接渲染

## 7. 常见问题

### 7.1 `cmake`、`g++`、`clang++` 找不到

先确认：

```powershell
where cmake
where g++
where clang++
where ninja
```

若当前 shell 未刷新 PATH，可临时执行：

```powershell
$env:Path = "F:\bin;C:\tools\winlibs-gcc-clang\mingw64\bin;$env:APPDATA\Python\Python314\Scripts;$env:Path"
```

### 7.2 MinGW 在中文路径下出现 filesystem 编码错误

当前项目路径包含中文目录名。MinGW 的 `std::filesystem` 在某些窄字符路径转换场景下可能报：

```text
filesystem error: Cannot convert character sequence: Illegal byte sequence
```

规避方案：

1. 优先使用 Web 后端调用，后端会尽量使用明确路径。
2. 或将项目复制到纯 ASCII 路径，例如：

```text
C:\work\next_project
```

3. 也可以改用 MSVC 工具链构建，但当前环境没有安装 MSVC `cl`。

### 7.3 Web 点击运行后提示找不到 `sim_dynamic_replay`

先构建：

```powershell
cmake --build cpp/build --target sim_dynamic_replay -j 4
```

确认文件存在：

```powershell
Test-Path cpp/build/sim_dynamic_replay.exe
```

### 7.4 端口被占用

更换端口：

```powershell
python -m uvicorn server:app --host 127.0.0.1 --port 8766
```

访问：

```text
http://127.0.0.1:8766
```

### 7.5 replay JSON 太大

降低：

- `max_sim_time`
- `repeat_count`
- 地图复杂度
- 前端默认只抽样渲染部分帧

## 9. 中期编队调控与报告命令

本节用于复现 2026-05-13 中期收口版本的关键结果。报告默认使用中文模板，并在仿真完成后输出 Markdown、CSV 与图片资源。

### 9.1 Python 编队调控消融

```powershell
python -m experiments.run_ablation --quick --scenario rrt_dual_channel_online --variant formation_aware_lookahead_adaptive --output-dir outputs/ablation_rrt_dual_channel
python -m experiments.run_ablation --quick --scenario formation_maze_stress_online --variant formation_aware_lookahead_adaptive --output-dir outputs/ablation_formation_maze_stress
```

重点查看：
- `outputs/ablation_rrt_dual_channel/report.md`
- `outputs/ablation_rrt_dual_channel/summary.csv`
- `outputs/ablation_formation_maze_stress/report.md`
- `outputs/ablation_formation_maze_stress/summary.csv`

这两组场景用于说明：
- 前瞻窗口可在狭窄通道和转弯前提前触发队形调整。
- RRT escape 可在窗口采样误判为死路时提供全局候选路径。
- `formation_aware_lookahead_adaptive` 相比普通 leader-only planner 能显式记录从机安全裕度、队形调整事件、阻塞原因和重规划接受结果。

### 9.2 C++ 报告输出

先构建 C++ 目标：

```powershell
cmake --build cpp/build --target sim_main sim_benchmark sim_warehouse sim_dynamic_replay --config Release
```

再运行 C++ 预设报告：

```powershell
python -m experiments.run_cpp_report --exe cpp/build/sim_warehouse.exe --title C++编队调控预设报告 -- --preset rrt_dual_channel_online --max-sim-time 0.5
```

当前 C++ 侧已能输出路径、飞行参数、碰撞/航点/重规划事件、编队调控元数据和 `formation_adaptation_events`。边界是：C++ 仍定位为可运行子集，不等同于 Python 主线的完整 RRT*/Informed RRT*、全量编队自适应和完整从机动力学复现。
