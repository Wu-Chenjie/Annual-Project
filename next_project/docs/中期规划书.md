### 1.3 已补齐的短期卡点

`tests/test_fault_tolerance_online.py` 原先只验证故障注入日志，却运行 12 秒在线容错场景，导致全量 pytest 超过 10 分钟仍未结束。现已将测试时间窗缩短到故障注入触发后的最小有效范围：

- `cfg.max_sim_time = 1.0`
- `cfg.planner_horizon = 1.0`
- `cfg.planner_replan_interval = 1.0`

该测试仍验证 `fault_log` 中存在 `type == "inject"` 的事件，并将单测耗时降至约 27 秒，使全量复现具备可接受的本机执行时间。

## 二、中期目标

中期阶段不再优先增加“新名词模块”，而是围绕论文和答辩最需要的证据链推进四类能力：

1. **连续轨迹优化闭环**：把离散路径点升级为时间化轨迹，补齐速度、加速度、jerk/snap、clearance 等指标。
2. **编队安全模块化**：把 `formation_safety` 从轻量工具函数推进为可复用、可测试、可被 C++ 子集消费的安全层。
3. **动力学参数可解释**：完善 `DroneParams` profile、标定说明和 profile 切换后的控制/风险提示。
4. **消融实验与图表流水线**：建立一键实验、指标提取、消融对照和 Markdown/CSV 报告输出。

中期 M2 的完成判据不是“实现所有高阶算法”，而是**每个关键研究主张都有可运行入口、可量化指标、可回退开关和可引用图表**。

## 三、任务分解

### M2-1 连续轨迹优化与指标闭环

**目标**：将当前 `trajectory_optimizer.py` 从轻量后处理推进为可量化的连续轨迹原型。

| # | 任务 | 交付物 | 验收标准 |
|---|---|---|---|
| 1.1 | 补齐轨迹指标提取 | `TrajectoryResult` 增加路径长度、RMS tracking、max jerk、mean jerk、jerk 平方积分、snap 平方积分 | `tests/test_trajectory_optimizer.py` 覆盖指标字段 |
| 1.2 | 增加 minimum-jerk/B-spline 原型 | `core/planning/trajectory_optimizer.py` | 已完成 minimum-jerk 原型与 `min_snap_proxy` 候选选择；B-spline 留作后续增强 |
| 1.3 | 接入离线场景对比 | `planned_trajectory` 输出扩展 | `warehouse` 和 `meeting_room` 已可输出轨迹指标 |
| 1.4 | 在线局部路径灰度接入 | 配置开关默认关闭 | 在线场景启用后无碰撞，耗时不超过基线 1.5 倍 |

**边界**：不在中期第一阶段承诺完整 GCOPTER/MINCO 复现；先做轻量、可回退、可解释的连续轨迹层。

### M2-2 formation_safety 模块化

**目标**：让编队安全链路从“散落在仿真循环里的检查”收敛为独立模块。

| # | 任务 | 交付物 | 验收标准 |
|---|---|---|---|
| 2.1 | 明确 safety API | `core/formation_safety.py` 文档化接口 | 已由 `test_formation_safety.py` 覆盖 min distance、downwash、deconflict、recovery |
| 2.2 | 抽取场景循环中的安全策略 | `ObstacleScenarioSimulation` 调用安全层接口 | 相关场景测试通过 |
| 2.3 | C++ 子集同步 | `cpp/include/formation_safety.hpp`、`cpp/src/formation_safety.cpp` | `test_cpp_sync_static.py` 守门通过 |
| 2.4 | 风险报告接入 | `risk_report` 读取 safety metrics | `test_risk_report.py` 通过 |

**边界**：Web dynamic replay 仍标注为 leader-centric 展示，不把其 follower proxy 当作完整编队安全证据。

### M2-3 DroneParams 标定与 profile 证据链

**目标**：让动力学参数从“能跑的默认值”变成“有来源、有约束、有适用范围”的 profile。

| # | 任务 | 交付物 | 验收标准 |

## 五、风险与缓解

| 风险 | 概率 | 影响 | 缓解 |
|---|---|---|---|
| 轨迹优化器引入耗时或数值不稳定 | 中 | 在线场景退化 | 默认关闭在线轨迹优化；所有优化结果必须通过 clearance gate |

### 8.4 答辩边界

答辩中应明确区分三层能力：

- Python 主线：当前中期创新和实验结论的主要依据。
