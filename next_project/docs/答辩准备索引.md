## 5. 答辩当天建议

开场按这个顺序讲：

1. 项目背景：室内多无人机编队飞行。
2. 难点：队形保持、避障、在线变化、故障容错。
3. 方法：动力学控制 + 多层规划 + APF/FIRI/GNN + 容错拓扑。
4. 论文映射：四篇论文分别落到四个层次。
5. 实验：多场景、误差、轨迹、日志、测试。
6. 局限与展望：实机验证、C++ 编译、MPC 时间化轨迹。

不要一上来讲公式。先讲问题和系统，再用公式支撑。

## 7. 中期收口答辩口径（2026-05-13）

### 7.1 先讲结论

当前版本可以作为中期验收版本提交。理由不是“功能很多”，而是每条核心主张都有可运行入口、可量化指标、可回退开关和可引用报告。

### 7.2 三个创新点

1. **formation-aware clearance 策略模块**：不只检查 leader 路径是否无碰撞，而是把从机包络和队形安全裕度纳入通道宽度判定。
2. **前瞻窗口触发队形变换**：在探测窗口内提前发现狭长通道和急转弯，避免到了障碍边缘才切换队形。
3. **RRT escape 与消融报告闭环**：当局部窗口误判为死路时，用 RRT escape 候选路径补充全局可达性，并在报告中记录 attempt/accepted/failed。

### 7.3 现场证据路径

| 主题 | 证据路径 | 现场说法 |
|---|---|---|
| 中期是否达标 | `docs/中期验收记录.md` 第 11 节 | 全量测试、C++ 构建、C++ 报告和跨线回归都有最新命令记录 |
| 编队调控 | `core/formation_adaptation.py`、`simulations/obstacle_scenario.py` | 已经从经验判断变成明确策略模块 |
| 消融对比 | `outputs/ablation_rrt_dual_channel/report.md`、`outputs/ablation_formation_maze_stress/report.md` | 可对比 leader-only、formation-aware、自适应前瞻三种模式 |
| C++ 对齐 | `tests/test_cpp_result_reporting.py`、`cpp/src/warehouse_main.cpp`、`cpp/src/dynamic_main.cpp` | C++ 是可运行子集，已补齐报告和事件字段 |
| 轨迹质量 | `outputs/ablation_m2_trajectory_snap/summary.csv` | snap 平方积分作为四旋翼高阶平滑性代理，jerk 只作辅助参考 |

### 7.4 必须主动说明的边界

- Python 主线最完整；C++ 是可运行子集，不宣称完整复现全部 RRT* / Informed RRT*。
- dynamic replay 是 leader-centric 动态障碍回放，不是完整多机动力学 benchmark。
- `minimum_jerk` / `min_snap_proxy` 是工程原型，不等于完整 GCOPTER/MINCO。
- DroneParams profile 是仿真参数证据链，不是实机标定结论。
