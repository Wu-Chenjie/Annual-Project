## 4. 答辩问答训练

最后读：

- `答辩QA预演.md`

建议练习方法：

1. 先看问题，不看答案，自己口头回答。
2. 再对照答案补关键词。
3. 每个回答控制在 30 秒到 90 秒。
4. 对高风险问题提前背熟短句。

高风险问题：

- GNN 是不是深度学习？
- 为什么 GNN 不是完整论文复现？
- 为什么 FIRI 是工程化版本？
- 为什么在线重规划不能覆盖任务航点？
- 为什么不用强化学习端到端控制？
- C++ 有没有编译验证？
- 项目最大局限是什么？

## 5. 答辩当天建议

开场按这个顺序讲：

1. 项目背景：室内多无人机编队飞行。
2. 难点：队形保持、避障、在线变化、故障容错。
3. 方法：动力学控制 + 多层规划 + APF/FIRI/GNN + 容错拓扑。
4. 论文映射：四篇论文分别落到四个层次。
5. 实验：多场景、误差、轨迹、日志、测试。
6. 局限与展望：实机验证、C++ 编译、MPC 时间化轨迹。

不要一上来讲公式。先讲问题和系统，再用公式支撑。

## 6. 一句话总括

本项目构建了一个可解释、可测试、可视化的多无人机室内编队仿真框架，把论文中的 APF 安全增强、GNN 双模式规划、风险调度思想和容错拓扑重构整合到同一条运行链路中。
