# 算法优化反效果归因分析

> 本文档复盘 `next_project` 在避碰、路径规划与控制器层面"优化后效果反而变差"的全部典型案例，给出根因与可复用的避坑准则。证据全部来自仓库内 `避碰优化技术文档.md`、`core/planning/` 与 `simulations/obstacle_scenario.py` 的实际迭代记录。

## 1. 反效果一览

| 轮次 / 优化点 | 优化意图 | 优化后指标 | 相对基线 |
|---|---|---|---|
| 第 2 轮：`discretize_margin = res·√3/2` 全局栅格膨胀 | 让薄管道在体素栅格中可见 | 碰撞 **1076 次**，A* 完全阻塞 | 357 → 1076，**+201%** |
| 第 2 轮：`discretize_margin = res·0.5` | 同上，弱化版 | A* 仍阻塞，规划航点数 = 0 | 失败 |
| 第 3 轮：迭代式薄障碍发现 + 重规划 | 局部修补栅格，绕过薄管道 | 航点完成率 **16%**，重规划崩溃 | 100% → 16% |
| 第 5/6 轮：排斥力 `max_repulsion = 8.0`（位置叠加） | 加强避碰排斥 | 航点完成率 **1%**，PID 跟踪失败 | 100% → 1% |
| 第 6 轮：领航机 `max_repulsion = 7.0` | 削弱影响范围 + 加大力 | 碰撞 **199 次**，从机碰撞复现 | 22 → 199 |
| 第 7 轮：`_inflate_r` 改为遍历调度队形取最大包络 | 兼顾 line/diamond 两种队形安全 | 栅格膨胀至 **18.7 m**，仓库整体不可通行 | A* 失败 |
| 第 8 轮：`_enforce_path_clearance` 校核乘数 `×1.15` | 提高安全裕度 | 路径变形过度 → 跟踪失败 | 无解 |
| 简单场景排斥力常驻 | 复用工业仓库参数 | 厘米级精度 **FAIL**（max=0.038m > 0.02m 阈值） | PASS → FAIL |

## 2. 根因聚类

### 2.1 R1 — 在全局状态上做了局部问题的修补

**典型案例：** 第 2 轮 `discretize_margin`、第 7 轮 `_inflate_r` 多队形包络。

**本质：** 问题是"少数薄管道在栅格里隐形"或"线性队形某瞬态半径大"，但修补手段是对**所有**障碍物 / **所有**调度队形同时放大。这把局部缺陷的修复代价转嫁到了全局的可通行性上。

**量化：** 仓库 ~25 个障碍物，每个再叠加 0.2–0.35 m 的体素膨胀，窄通道（实际宽 ~4 m）被全部填满。线性队形 spacing=6 m 时 envelope_radius=18.2 m，加上 arm 与 safety_margin 后栅格膨胀 18.7 m，约等于整个仓库的横向尺寸。

**避坑准则：** 全局参数只能用来表达**全局共享**的物理约束（如安全裕度、机体半径）。对"少数异常对象"的修复要落到对象级（如 SDF-aware 的 per-voxel 精判，第 9 项修复），不要进全局栅格膨胀。

---

### 2.2 R2 — 离散结构上"打补丁"产生级联放大

**典型案例：** 第 3 轮迭代式薄障碍发现 — 「发现一个 → 标记一个 → 重规划」。

**本质：** 每一轮发现都向栅格再写入一批"虚拟占据"，下一轮 A* 在更紧的栅格上搜索 → 容易再次发现新的不安全段 → 进一步膨胀。这是缺乏全局视角的贪心闭环，最终在密集障碍场景下让栅格收敛到不可通行状态，A* 回退到直线，碰撞反而暴增。

**避坑准则：迭代式修补必须有终止条件与全局上界**：要么开放空间预算（如剩余空闲体素比例下界），要么改用一次性两阶段（栅格保证基本通行 + 后处理精修，即第 4 轮 SDF 校核思路）。

---

### 2.3 R3 — 把"力 / 修正量"叠到了错误的物理层

**典型案例：** 第 5/6 轮排斥力以"位置偏置 (m)"形式叠加到 `target_pos`。

**本质：**
1. 量纲混淆：方法被注释为"m/s² 等效"但实际单位是 m。
2. PID 位置环把瞬时跳变到 5–8 m 远处的目标当作命令，控制器进入饱和或大幅震荡。
3. 力越大跟踪越差，**安全 vs 跟踪形成硬约束**（`max_repulsion > 5.0` 时航点完成率断崖到 1%）。

修复（第 11.2 节修复 3）将语义改为"前馈加速度 m/s²"，注入到 `compute_control(..., target_acc=...)`，位置环不再被扰动，约束自然解除，max_repulsion 可放大到 8.0 仍保持 100% 完成率。

**避坑准则：** 在控制层叠加修正量前先确认它的**量纲、注入点、对位置环/速度环/加速度环的因果链**。"加到目标上"听起来等价，物理上不等价。

---

### 2.4 R4 — 复用了不同场景下被默认启用的全局开关

**典型案例：** 简单 obstacle 场景精度退化。

**本质：** 排斥力影响距离 `5×collision_margin = 2.5 m`，简单场景房间总宽不过几米，从机时刻处于墙壁排斥力影响范围内 → 控制器被持续推离编队中心 → max 误差 0.038 m，超过 0.02 m 阈值。仓库场景"开就是收益"，简单场景"开就是退化"。

**避坑准则：** 默认全局启用的避碰行为，必须在场景配置层做**场景相关的开关或降权**（如简单场景把 max_repulsion 降至 0 或 influence_distance 降至 1×margin）。

---

### 2.5 R5 — 单参数误差掩盖了多参数耦合

**典型案例：** 第 6 轮领航机参数搜索出现非单调（`5.0→6.0→7.0` 时碰撞 22→41→199）。

**本质：** `max_repulsion` 增大的同时 `influence_distance` 缩小，且从机参数同步联动；这构成 6 维耦合空间（领航 / 从机 × max_repulsion × influence × 校核迭代/步长）。线性单参数试探在耦合空间里会出现非单调和虚假最优。

**避坑准则：** 多参数避碰系统应当在配置层固化耦合关系（如把 `influence` 设为 `max_repulsion` 的函数，或固定 `influence` 仅搜索 `max_repulsion`），并优先使用网格搜索 / 贝叶斯优化，避免凭直觉一次只动一个量。

---

### 2.6 R6 — 路径航点级安全 ≠ 连续轨迹安全

**典型案例：** 第 4 轮 SDF 校核 — 494/494 航点 SDF ≥ 0.5 m，但碰撞仅从 357 降到 326。

**本质：** PID 控制器在航点间产生连续轨迹，最大瞬态偏离可达 0.4 m，足以切入薄管道附近的危险带。"航点全部安全"只是**离散采样意义**上的安全。

**避坑准则：** 任何对离散路径的安全证明必须配套一个"轨迹级"防护，在本项目里是第 2 层线段插值 + 第 3 层实时排斥的组合。单层校核必然有漏洞。

---

### 2.7 R7 — 物理对象的对称性假设错误

**典型案例：** 第 7 轮多队形 `_inflate_r`。

**本质：** 把 envelope_radius 定义为 `max(|offset|)`，对线性纵向延伸队形（offset = (-6,0,0), (-12,0,0)…）会被错误解读为"横向也膨胀 12 m"。线性队形的横向包络其实只有 0 m。

**避坑准则：** 包络计算应分轴或分方向（横向 / 纵向），不要用各向同性范数代替方向相关的物理量。

---

### 2.8 R8 — 算法实现细节不到位时优化变成噪声放大

**典型案例：**
- RRT* rewire 后子树代价未传播 → 渐近最优性被破坏，节点越多结果越随机；
- Informed RRT* 球面采样用 `θ ~ U(0, π)` → 两极采样过密；
- D* Lite "惰性删除" → 堆条目随时间无界增长；
- Catmull-Rom 端点切线 `(p2-p0)/2 = (p2-p1)/2` → 端点处切线幅值减半，平滑后路径偏离栅格中心。

这些都属于"理论正确但实现错位"。在小规模/短时长 demo 里看不出问题，节点 / 障碍物 / 时长一上来，"优化版"就开始系统性劣于基线。

**避坑准则：** 对教科书算法，先把实现的数学正确性单测过关（如 rewire 后任意子孙节点 cost 等于父链路径长度、采样分布卡方检验），再上规模实验。

## 3. 决策清单（实操）

在评估"某个优化是否值得合入"时，按下列顺序自查：

1. **量纲检查** — 新加项的物理单位是否与注入点的目标变量一致？
2. **作用域检查** — 这是局部缺陷还是全局缺陷？局部问题禁止用全局参数修。
3. **饱和与可跟踪性** — 修正量上限是否超过控制器在 `dt` 内可吸收的能力？
4. **离散 vs 连续** — 安全性是航点级还是轨迹级？两者都要保证。
5. **跨场景副作用** — 默认开启的逻辑在最简单场景下是否仍中性？
6. **耦合参数** — 新参数是否与既有参数耦合？耦合关系是否被固化到配置层？
7. **算法实现单测** — 教科书算法的关键不变量（最优性 / 分布均匀性 / 内存有界）是否有冒烟测试？
8. **回退开关** — 是否提供 `enable_xxx` 配置能一键退回基线？

### 5.2 R2（迭代补丁级联放大）的解决方法

**方案：用"两阶段一次性"替代"贪心迭代"**

```
阶段 A — 栅格层（一次性）：保证基本可通行
  to_voxel_grid() 用最小膨胀 = arm_length + safety_margin
  不再尝试在栅格里捕获薄障碍物（交给 SDFAware）

阶段 B — 后处理层（一次性）：SDF 校核 + 线段插值
  路径生成后做一次 _enforce_path_clearance
  不在循环里改栅格
```

**终止条件设计**（如未来仍需迭代）：

```python
def iterative_repair(grid, max_iter=3, free_ratio_floor=0.4):
    for it in range(max_iter):
        free_ratio = (grid.data == 0).sum() / grid.data.size
        if free_ratio < free_ratio_floor:
            log.warning(f"iter {it}: free_ratio={free_ratio:.2f} < floor, abort")
            break
        ...
```

强制：剩余空闲体素比例下界 `free_ratio_floor=0.4`，迭代上限 `max_iter=3`。

**落点：** `simulations/obstacle_scenario.py::_setup_obstacles` 注释中明确禁止"边发现边膨胀"模式。

### 5.5 R5（多参数耦合）的解决方法

**方案 1（短期）：在配置层固化耦合关系**

```python
@dataclass
class RepulsionParams:
    max_repulsion: float
    influence_distance: float = field(init=False)
    collision_margin: float = 0.5

    def __post_init__(self):
        # 影响距离与最大力联动：力越大影响越远
        self.influence_distance = self.collision_margin * (3.0 + 0.4 * self.max_repulsion)
```

只搜索一个自由参数 `max_repulsion`，避免独立调参产生非单调。

**方案 2（中期）：网格搜索 + 帕累托前沿**

新增脚本 `simulations/sweep_repulsion.py`：

```python
grid = {
    "leader_max_repulsion": [4, 6, 8, 10],
    "follower_max_repulsion": [3, 5, 7],
}
# 输出 Pareto(碰撞数, 平均误差) 前沿，自动选择前沿上 collision==0 的最低误差点
```

**方案 3（长期）：贝叶斯优化**

引入 `scikit-optimize` 在 6 维参数空间做 50 轮贝叶斯优化，目标函数 `J = collision_count + 10 × mean_error_overflow`。

**落点：** 新增 `simulations/tuning/` 子目录，放 sweep / bayes 脚本与结果 JSON。

### 5.6 R6（航点级安全 ≠ 轨迹级安全）的解决方法

**方案：三层防护明确分工，每层单独评估**

```
L1 路径层：A* + SDF 校核   → 指标：所有航点 SDF ≥ min_clearance
L2 轨迹层：线段插值 + 重采样 → 指标：插值采样点 SDF ≥ min_clearance × 0.8
L3 控制层：实时排斥前馈     → 指标:在线 min(SDF) > 0
```

**评估脚本：** `tests/test_three_layer_guard.py`，分别关闭 L2 / L3 跑仓库场景，验证：

| 关闭 | 期望碰撞数 |
|---|---|
| 关闭 L3 | 显著上升（验证 L3 必要性） |
| 关闭 L2 | 中等上升（验证 L2 必要性） |
| 关闭 L1 | 灾难（验证 L1 必要性） |

确保每一层都"无法被另两层完全替代"，否则可以删掉该层简化系统。

**升级方向：** L2 由"等距线段采样" 升级为"按轨迹动力学预测采样"——根据领航机当前速度/加速度向前预测 0.5 s 轨迹再做 SDF 校核。

**落点：** `simulations/obstacle_scenario.py::_predict_trajectory_clearance` 新方法。

### 5.7 R7（包络计算各向同性误用）的解决方法

**方案：分轴包络 + 形态学膨胀**

```python
def envelope_per_axis(formation_offsets: list[np.ndarray]) -> tuple[float, float, float]:
    """返回 (lateral, longitudinal, vertical) 三轴包络半径"""
    arr = np.stack(formation_offsets)  # (N, 3)
    return float(np.max(np.abs(arr[:, 0]))), \
           float(np.max(np.abs(arr[:, 1]))), \
           float(np.max(np.abs(arr[:, 2])))


def _inflate_r_per_axis(self) -> tuple[float, float, float]:
    lat, lon, vert = envelope_per_axis(self._initial_offsets())
    safe = self.safety_margin + 0.2
    arm = 0.2
    return lat + arm + safe, lon + arm + safe, vert + arm + safe
```

**栅格膨胀使用 `scipy.ndimage.binary_dilation` 配合椭球结构元**（不是球），即可分轴施加不同的膨胀厚度。线性队形纵向膨胀大、横向膨胀小，仓库窄通道得以保留。

**落点：** `simulations/obstacle_scenario.py::_inflate_r`、`core/obstacles.py::OccupancyGrid.inflate(rx, ry, rz)` 三轴版本。

### 5.8 R8（教科书算法实现错位）的解决方法

**方案：对每个教科书算法定义"不变量单测"**

| 算法 | 不变量 | 单测 |
|---|---|---|
| RRT* rewire | 任意节点 v 的 `cost[v] == 沿 parent 链回溯到 root 的累计欧氏距离` | `tests/test_rrt_rewire_invariant.py` |
| Informed RRT* 椭球采样 | 1000 次采样在椭球内的方向分布通过 χ² 均匀性检验（p > 0.05） | `tests/test_informed_uniform.py` |
| D* Lite 堆 | 每次 `compute_shortest_path` 后 `len(U) ≤ 节点数` | `tests/test_dstar_heap_bound.py` |
| Catmull-Rom 端点 | `||tangent(s=0)|| > 0.5 × ||tangent(s=0.5)||` | `tests/test_catmull_endpoint.py` |

## 6. 待防范的潜在冲突（来源：`GNN分层双模式架构设计.md` 评审）

本节记录在评审 [GNN分层双模式架构设计.md](GNN分层双模式架构设计.md) 时识别出的、与第 2 节既有反效果根因（R1–R8）直接冲突的设计意图。**这些条款在 GNN 分层双模式落地前必须修补**，避免历史教训重演。

### 6.1 C1 — 编队整体势场"替代"逐机独立 APF（违反 R6）

**冲突点：** 设计文档 §2.1 提议用 `FormationAPF.compute_formation_avoidance` 以编队重心计算斥力并分配给从机，"替代当前的逐机独立 APF"。

**与 R6 冲突：** R6 已论证航点级 / 编队级安全 ≠ 轨迹级 / 单机级安全。L3 实时排斥层（APF）必须由每架从机基于自身位姿独立感知；以重心为代理点会让贴障碍物一侧的从机感受不到斥力，等同于关闭 L3 防护。

**修补准则：** 编队整体势场只能**叠加**到独立 APF 之上，不得替代：

```
F_total[i] = F_centroid (编队整体) + F_relative[i] (队形保持) + F_individual[i] (单机独立 APF, L3)
```

整体势场权重 `α∈[0,1]` 由场景配置；`α=0` 时退化为现有逐机独立 APF。

---

### 6.2 C2 — 通信约束力的量纲未声明 + 阈值阶跃（违反 R3）

**冲突点：** 设计文档 §2.3 给出 `F_comm = k_comm * (dist/comm_range) * toward_other`，注释"k_comm = 0.3"未标注量纲；且 `dist > 0.8*comm_range` 时引力突现，存在阶跃。

**与 R3 冲突：** R3 是项目史上最严重的反效果根源（max_repulsion 量纲混淆使航点完成率 100%→1%）。任何新增"力 / 修正量"必须先固化注入点与量纲。

**修补准则：**

1. 显式标注 `# 量纲: m/s²，作为 target_acc 前馈注入，禁止叠加 target_pos`，并加运行时断言 `assert ||F_comm|| ≤ k_comm + 1e-6`。
2. 用平滑过渡替代阶跃：

   ```python
   gate = 0.5 * (1 + tanh((dist/comm_range - 0.8) / 0.05))
   F_comm = k_comm * gate * (dist/comm_range) * (diff/dist)
   ```

3. `tests/test_comm_force_dim.py` 验证返回值范围与量纲注释一致。

---

### 6.3 C3 — FaultDetector 阈值硬编码（违反 R4）

**冲突点：** 设计文档 §5.3 中 `max_jerk=10.0、pos_dev_threshold=5.0、saturate_steps=50` 全部硬编码在 `FaultDetector.__init__`。

**与 R4 冲突：** R4 案例（简单 obstacle 场景复用仓库 `max_repulsion` 致厘米精度退化）证明：默认全局启用的检测/避碰逻辑在简单场景下会产生跨场景副作用。故障检测在轻载场景中可能因风扰被误触发，引发不必要的拓扑重构。

**修补准则：**

1. 阈值字段全部并入 `SimulationConfig`：

   ```python
   fault_detector_max_jerk: float = 10.0
   fault_detector_pos_dev: float = 5.0
   fault_detector_saturate_steps: int = 50
   ```

2. 在 `_config_basic` / `_config_obstacle` 等简单预设里默认 `fault_detection_enabled=False`。
3. `tests/test_scene_regression.py` 增加"无故障运行 30s 不应触发误检"的回归断言。

---

### 6.4 C4 — n_decay 自适应每步采样（性能反效果伏笔）

**冲突点：** 设计文档 §2.2 在 `_obstacle_repulsion` 中调用 `_estimate_local_obstacle_density`，需要在 r_rep 球内采样 SDF。

**潜在反效果：** dt=0.012s × N 架机 × M 个障碍物的高频调用，每次都做球形采样会显著拉长仿真步长，可能让在线模式的 `WindowReplanner.interval` 实际无法满足。这不属于既有 R1–R8，但属于"优化引入新瓶颈"的同类风险。

**修补准则：**

1. 缓存策略：当前位置距上次采样点 < r_rep/4 时复用密度估计；
2. 控制频次：每 K=10 步重算一次密度；
3. 验收增加性能门槛：`tests/test_apf_perf.py` —— 启用自适应 n_decay 后单步耗时不超过原值 1.2 倍。

---

### 6.5 C5 — 多 APF 开关组合的耦合（违反 R5）

**冲突点：** 设计文档 §7.1 暴露 4 个独立开关 `apf_adaptive_n_decay / apf_formation_centroid / apf_comm_constraint / apf_rotational_escape`，组合空间 16 种。

**与 R5 冲突：** R5（max_repulsion 与 influence_distance 联动产生非单调）已证明独立调参在耦合空间里出现虚假最优。4 个 APF 开关共享同一斥力输出，必然耦合。

**修补准则：**

1. 在配置层固化为预设档位：

   ```python
   apf_paper1_profile: Literal["off", "conservative", "aggressive"] = "off"
   # off:          四开关全 False（基线）
   # conservative: n_decay + 旋转力场（仅风险最低的两项）
   # aggressive:   全开（含编队整体势场 + 通信约束）
   ```

2. 三场景回归（basic/obstacle/warehouse）分别验证三档 profile 无指标劣化；
3. 仅在确有需要时才暴露独立开关给开发模式。

---

### 6.6 C6 — GNN/旋转力场公式复现的严谨性（新增类别 R9 — 论文复现错位）

**冲突点：**

1. 论文2 的 GNN 公式 19-20 原本作用于栅格神经元上的稠密活动场；设计文档 §3.3 把它直接套到稀疏可见图节点上，是工程改写而非论文复现。
2. 设计文档 §2.4 旋转力场写作 `cross(total_repulsion, cross(rep_dir, to_goal_dir))`，根据 BAC-CAB 恒等式 `a×(a×b) = (a·b)a − (a·a)b`，结果几乎平行于 `to_goal`，等价于一个朝目标的引力，与论文1"切向旋转力打破势场对称"语义不符。

**新增根因 R9（论文复现错位）：** 教科书/论文公式被搬到不同的载体（神经元→图节点）或写错代数变换时，"按论文实现"的合规外观会掩盖"行为不等价"的实质。R9 与 R8（实现错位）的区别在于 R9 源头是论文向工程的转译，R8 源头是教科书算法本身的实现细节。

**修补准则：**

1. **GNN 可见图变体**：在设计文档显式声明"本项目对论文2 GNN 的可见图变体（vertex-only diffusion）"，并在 M2 验收基线里同时跑"论文 2 原始栅格 GNN" + "本项目可见图 GNN"，对比路径长度差异；
2. **旋转力场公式**：改用 Rodrigues 旋转 `rep_dir`（绕 `rot_axis` 旋转 θ ≈ π/2），而不是再做一次 cross：

   ```python
   F_rot = mu_escape * (rep_dir * cos(θ) + cross(rot_axis, rep_dir) * sin(θ))
   ```

3. **不变量单测**：
   - `tests/test_gnn_steady_monotone.py`：GNN 收敛后活动场从目标向外单调递减；
   - `tests/test_rotation_force_tangent.py`：旋转力场与 `total_repulsion` 内积近 0（即真正的切向力）。

### 6.7 C7 — 一次性合入 4 篇论文 9 个文件的规模风险（横切）

**冲突点：** 设计文档 §八 计划 7.5 人日内一次性引入论文 1–4 的全部改动。

**与历史经验冲突：** §2.8 R8 与第 5.10 节优先级建议都强调"小步快跑、单测先行"。一次性合入 9 个文件（其中 4 个新增、5 个修改）将产生大量耦合点，回归测试爆炸。

**修补准则：**

1. **拆为两批 PR**：
   - **PR-A**（论文 2 + 论文 3 风险调度）：M1/M2/M5/M7 部分 —— 双模式 + GNN + 自适应间隔；
   - **PR-B**（论文 1 + 论文 4）：M3/M4/M6 —— APF 增强 + 容错重构。
2. PR-A 与 PR-B 之间必须跑完三场景回归（basic/obstacle/warehouse），并由 `tests/test_scene_regression.py` 守门；
3. 每个新算法在合入前必须有至少 1 项不变量单测（参照 §5.8 表）。
