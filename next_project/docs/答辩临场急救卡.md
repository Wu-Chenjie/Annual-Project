# 答辩临场急救卡

> 用法：答辩前 5 分钟只看这一页。被严厉追问时，先稳住边界，再给证据，不要争辩式硬撑。

---

## 1. 开场 20 秒定调

本项目是一个面向室内复杂环境的多无人机编队仿真与报告系统。当前中期主线不是声称完成实机安全证明，而是完成了“编队感知安全裕度、前瞻队形调整、RRT escape 候选路径、消融实验和中文报告”的可运行闭环。

---

## 2. 三个创新点

1. **formation-aware clearance**：规划时不只看 leader 轨迹，还把从机包络和队形宽度纳入安全裕度。
2. **前瞻窗口队形调整**：在进入狭窄通道或大角度转弯前提前变换队形，而不是到障碍物边缘才反应。
3. **RRT escape + 消融报告闭环**：当局部窗口误判为死路时，用 RRT 生成候选绕行路径，并记录 attempt/accepted/failed，最后通过 CSV、Markdown 和图片报告复核。

## 3. 最常见狠问题与短答

### 问：这是不是只是调参？

不是。调参不会改变决策结构。本项目的变化是：路径验收从 leader-only 变成 formation-aware；队形切换从到点反应变成前瞻触发；局部窗口死路从直接失败变成 RRT escape 候选补充。参数可以换，但这三条机制仍然成立。

### 问：你证明算法安全吗？

当前不是形式化安全证明，而是工程级安全 gate。它能基于离散路径、障碍物距离、队形包络和安全裕度减少碰撞风险，但不等于连续时间可达性证明。后续可以接控制屏障函数、可达集或 MPC。

### 问：为什么不用 jerk 平方积分作为最优判据？

四旋翼不是简单二阶质点系统。jerk 可以作为辅助平滑性指标，但 snap 更接近四旋翼位置轨迹到姿态和控制输入变化的高阶平滑需求。当前 `trajectory_snap_squared_integral` 是代理指标，不是完整 GCOPTER/MINCO。

### 问：C++ 和 Python 完全一致吗？

不完全一致。Python 是研究主线；C++ 是可运行子集。C++ 已对齐 preset、报告字段、事件输出、障碍物建模和编队调控元数据，但不宣称完整复制 Python 的全部 RRT*/Informed RRT* 和完整从机动力学。

### 问：两个地图能证明泛化吗？

不能证明统计泛化。两个地图是压力场景，用于验证机制是否成立：双通道图验证局部假死路和 RRT escape，迷宫压力图验证狭窄通道、转弯和队形提前调整。泛化需要后续多 seed、随机地图和更长仿真。

### 问：GNN 是深度学习图神经网络吗？

不是。本项目中的 GNN 是神经动力学启发的可见图活动扩散变体，不是训练式 Graph Neural Network，因此不需要训练数据集。
