# 照片采集操作手册

本文档只解决一个问题：为了让当前 `next_project` 的 `COLMAP + OpenMVS -> 地图 JSON` 自动化链路跑通，室内照片应该怎么拍。

如果拍得不对，常见结果不是“地图有点差”，而是直接重建失败，例如：

- 只上传 1 张照片
- 多张照片之间没有重叠
- 画面里全是白墙、玻璃、镜子
- 走位混乱，COLMAP 找不到稳定匹配

## 1. 最低要求

当前系统的硬门槛和建议值：

- 最少 `2` 张照片才能启动重建
- 实际可用建议：`10` 到 `20` 张
- 相邻照片重叠建议：`60%` 到 `80%`
- 一条走廊、一个房间、一个实验角落，尽量用连续路径拍完整

一句话理解：不要拍“几张互相无关的图片”，要拍“一条连续观察轨迹”。

## 2. 正确拍法

### 2.1 房间拍法

适合会议室、办公室、小实验室。

拍摄步骤：

1. 站在房间一角，先拍第一张。
2. 横向移动一点，再拍第二张。
3. 保证第二张里还能看到第一张中的大部分家具、墙角、门框。
4. 按同样方式沿房间边缘缓慢移动，绕房间拍一圈。
5. 如果房间中央有桌子、设备、货架，再补一圈近距离照片。
6. 最后补拍高处和低处细节，例如空调下方、桌腿、设备底座、门槛。

核心原则：

- 不是原地转一圈乱拍
- 不是每次都拍完全不同方向
- 每移动一步，都要让前一张和后一张“看见同一批东西”

### 2.2 走廊拍法

适合学校走廊、公司通道、狭长实验区。

拍摄步骤：

1. 沿走廊一侧缓慢前进。
2. 每隔 `0.5` 到 `1.5` 米拍一张。
3. 镜头略微朝前，不要完全垂直于墙面。
4. 每拍一张，画面里都要保留上一张中的门、窗、灯、地砖缝、消防箱等重复特征。
5. 到转角处时，多拍几张过渡图，让转角前后视野有重叠。

核心原则：

- 走廊最怕“每张都像纯白长廊，没特征”
- 门框、地砖、标识牌、灯具这些重复结构非常重要

### 2.3 设备区 / 实验台拍法

适合有桌子、机柜、箱体、管线的区域。

拍摄步骤：

1. 先远距离拍一圈，建立整体结构。
2. 再围绕关键设备补近距离照片。
3. 让设备与背景同时出现，不要只拍设备特写。
4. 对细长结构，例如管线、支架、桌腿，尽量从多个侧面拍。

核心原则：

- 先有整体，再有局部
- 不要只有局部特写，否则系统只知道“物体表面”，不知道“物体在空间中的位置”

## 3. 错误拍法

下面这些拍法很容易让当前自动化流失败：

- 只拍 `1` 张或 `2` 张完全不同内容的照片
- 原地转一圈，但每张都看不见前一张内容
- 全部照片都是白墙、玻璃门、镜子、反光地板
- 边走边拍但抖动严重、模糊明显
- 一张横屏、一张竖屏、一张歪斜，姿态变化过大
- 只拍天花板或只拍地面
- 大量人经过、门在开关、屏幕内容在变化

一句话判断：如果你自己连续翻照片，都很难看出它们是同一个空间的连续视角，重建大概率也会失败。

## 4. 拍摄时具体注意什么

### 4.1 重叠比

每张照片与前一张至少保留大部分共同内容。

简单判断方法：

- 下一张里，前一张中最重要的家具、门框、墙角、设备，至少还能看到一半以上

### 4.2 光照

- 尽量在稳定光照下拍
- 不要一会儿背光一会儿强反光
- 避免强曝光和大面积过暗

### 4.3 清晰度

- 手不要抖
- 不要边快走边拍
- 拍完随手放大看一眼，确认门框、桌边、设备轮廓不是糊的

### 4.4 画面内容

优先保留这些东西：

- 门框
- 墙角
- 桌椅边缘
- 货架
- 管线
- 标识牌
- 地砖缝
- 灯具
- 柜体棱边

这些都是几何和纹理特征，COLMAP 比较容易匹配。

### 4.5 尺度参照

如果后面需要更可信的地图比例，拍摄时最好让已知尺寸物体进入画面，例如：

- 标尺
- A4 纸
- 已知尺寸地砖
- 标准门宽
- 已知尺寸桌面

这样后面更容易判断导入地图后的 `scale` 是否合理。

## 5. 当前项目的推荐最小方案

如果你只是想先把 Web 自动化流跑通，按下面这个最低可用方案做：

### 5.1 小房间

- 总照片数：`10` 到 `12` 张
- 房间四周绕一圈：`8` 张左右
- 中间设备或桌子补拍：`2` 到 `4` 张

### 5.2 走廊

- 总照片数：`12` 到 `20` 张
- 每隔 `0.5` 到 `1.5` 米拍一张
- 转角额外补 `2` 到 `4` 张

### 5.3 单个设备区

- 总照片数：`8` 到 `15` 张
- 先外圈 `6` 到 `8` 张
- 重点结构补 `2` 到 `6` 张

## 6. 拍完后怎么自检

在上传到 Web 之前，先自己做一次人工检查。

### 6.1 三个必须满足的问题

1. 这些照片是不是同一个空间的连续观察过程？
2. 任意相邻两张，是不是都能看到明显相同的家具/墙角/设备？
3. 照片里是不是有足够多非白墙、非玻璃的结构特征？

三个里只要有一个回答不稳，就不要急着上传，先补拍。

### 6.2 一分钟快速筛图法

- 删除明显模糊的图
- 删除几乎纯白墙的图
- 删除重复度极高但更模糊的图
- 保留视角连续、内容稳定的图

## 7. 上传到当前系统时怎么做

当前系统是自动化流，不需要手工敲完整重建命令。

步骤：

1. 打开 `http://127.0.0.1:8765`
2. 在“照片建模”里填写场景名
3. 一次选中一组连续照片
4. 点击“一键重建为地图”
5. 页面会自动显示重建状态

如果系统直接报错：

- `Reconstruction needs at least 2 photos`
  说明你上传的照片数量不够
- `COLMAP did not produce dense/0/sparse cameras output`
  说明常见原因是照片太少、重叠不足、纹理太差或模糊太严重

## 8. 最后一句建议

对于当前这套链路，拍照时最重要的不是“好看”，而是“连续、稳定、重叠、特征多”。

如果你只能记住一句话，就记这个：

`沿着空间慢慢走，连续拍，让前后两张总能看到同一批门框、墙角、桌椅和设备。`
