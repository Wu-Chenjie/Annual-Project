## 0. 先搞清楚：我们到底在解决什么问题？

### 0.1 四旋翼是什么？

想象一个**十字架**，四个端点各装一个螺旋桨：

```
    旋翼1 (逆时针)
       ↑
旋翼2 ← ● → 旋翼4 (逆时针)
(顺时针)  ↓
    旋翼3 (顺时针)
```

**关键原理**：
- 四个旋翼同时加速 → 飞机**上升**
- 旋翼1、3加速，2、4减速 → 飞机**俯仰**（往前/后倾）
- 旋翼2、4加速，1、3减速 → 飞机**滚转**（往左/右倾）
- 旋翼1、3与2、4产生相反扭矩 → 飞机**偏航**（左右转头）

四旋翼有**4个输入**（四个旋翼转速），但能控制**6个运动**（前后、左右、上下、俯仰、滚转、偏航）。输入比输出少，是因为**姿态和水平移动是耦合的**：你先倾斜机身，推力的水平分量就带着你飞。

### 0.2 为什么需要"编队"？

一架无人机能干的事有限。比如：
- 搜索救援：多架同时搜索大片区域
- 军事侦察：三角队形互相掩护
- 农业植保：多架一起喷洒效率更高

**编队核心问题**：每架无人机要知道自己在队形中的"相对位置"，不能互相撞上，还得一起避开障碍物。

### 0.3 为什么需要"路径规划"？

你在迷宫里走路，需要**提前想好走哪条路**才能不撞墙。无人机也一样：
- 从A点飞到B点，中间有墙、柱子、其他飞机
- 不能"走到哪看到哪再躲"，反应来不及
- 需要**预先计算一条安全的路线**

### 0.4 为什么需要"仿真"？

真飞机试飞很贵，摔了更贵。在电脑里先模拟：
- 测试控制器能不能稳住飞机
- 测试多架飞机会不会相撞
- 测试遇到风、故障时会不会坠毁

想象你要做一个乐高机器人。你有积木块（零件）、说明书（组装指南）、测试记录（检查装得对不对）。代码也是这样分的：

| 文件/文件夹 | 作用 | 类比 |

| `allocator.py` | 把"我想上升+前倾"翻译成四个旋翼各自转多快 | 意图→动作翻译 |
| `planner/` | 各种找路算法 | 不同品牌的GPS导航 |
| `topology.py` | 管理多架飞机的队形 | 舞蹈队队形编排 |
| `fault_detector.py` | 检测哪架飞机出故障 | 飞机黑匣子报警 |
| `simulations/` | 把计算结果画成动画/图表 | 录像回放 |

### 1.2 运行流程

```

        ↓
```

## 2. 四旋翼动力学模型

### 2.1 什么是"状态"？

一架飞机在天上，完整描述它需要12个数字：

| 变量 | 含义 | 例子 |
|------|------|------|
| $p_x,p_y,p_z$ | 位置（在哪） | (10, 20, 5) 米 |
| $v_x,v_y,v_z$ | 速度（飞多快） | (2, 0, -1) 米/秒 |
| $\phi$ (phi) | 滚转角（左右倾斜） | 0.1 rad（约6°） |
| $\theta$ (theta) | 俯仰角（前后倾斜） | -0.05 rad（约-3°） |
| $\psi$ (psi) | 偏航角（机头朝向） | 1.57 rad（朝东） |
| $p,q,r$ | 角速度（倾斜变化多快） | (0, 0, 0.5) rad/s |

**为什么需要12个？** 物理定律告诉你：只知道"在哪"不够，还得知道"跑多快"才能预测下一秒在哪。这就是牛顿力学。

**答辩话术**："状态向量采用12维欧拉角表示，包含位置3维、速度3维、姿态角3维和角速度3维，完整刻画四旋翼的6自由度刚体运动。"

### 2.2 平动：飞机怎么移动？

牛顿第二定律：$F = ma$

#### ① 重力（总是向下）

$$F_g = -mg$$

$m$ 是飞机质量（2kg），$g$ 是重力加速度（9.8 $m/s^2$），负号表示向下。

#### ② 旋翼推力（垂直于机身向上）

飞机倾斜了，旋翼推力不是直直向上。需要一个"方向转换器"把"机身里的方向"转成"地面坐标系里的方向"——这就是**旋转矩阵** $R(\phi,\theta,\psi)$。

推力在地面坐标系里：

$$F_{\text{thrust}} = R \cdot \begin{bmatrix}0\\0\\T\end{bmatrix}$$

$T$ 是四个旋翼的总推力。旋转矩阵就像把"机身上方"这个方向，转到"地面坐标系中实际指向哪"。

#### ③ 空气阻力

$$F_d = -k_d \|v - v_w\| (v - v_w)$$

拆解：
- $v$：飞机速度
- $v_w$：风速
- $v - v_w$：飞机相对于风的速度
- $\|v - v_w\|$：相对速度的大小
- 阻力方向和相对速度**相反**，大小和相对速度的**平方**成正比

**合起来**：

$$m \cdot \dot v = F_{\text{thrust}} + F_d + F_g$$

左边 $\dot v$ 是加速度，右边是所有力的和。

**答辩话术**："平动动力学考虑了推力、重力和非线性空气阻力。旋转矩阵将机体系推力映射到惯性系，阻力采用与相对风速平方成正比的非线性模型，能够反映真实空气动力学特性。"

### 2.3 转动：飞机怎么倾斜？

玩过那种抽一下能立在手上的陀螺吗？快倒下时推它一下，它不是直接倒，而是往侧面歪——这就是**陀螺效应**。

四旋翼四个旋翼都是高速旋转的陀螺。飞机本身转动时，旋翼旋转和飞机自转会互相影响。

**欧拉方程**：

$$I \dot \omega = \tau - \omega \times (I\omega) + \tau_g$$

逐个拆解：

| 项 | 含义 | 类比 |
|----|------|------|
| $I$ | 转动惯量 | 物体的转动"质量" |
| $\omega$ | 角速度 | 陀螺转多快 |
| $\dot\omega$ | 角加速度 | 陀螺加速/减速 |
| $\tau$ | 外部力矩 | 你推陀螺的力 |
| $\omega\times(I\omega)$ | **陀螺耦合项** | 快倒的陀螺往侧面歪 |
| $\tau_g$ | **旋翼陀螺反力矩** | 直升机尾巴要装小螺旋桨抵消扭矩 |

**旋翼反力矩**：

四个旋翼中，1、3顺时针转，2、4逆时针转（互相抵消，飞机不会自己打转）。定义净转速：

$$\Omega_{\text{net}} = \Omega_1 - \Omega_2 + \Omega_3 - \Omega_4$$

如果转速完全一样，$\Omega_{\text{net}}=0$。想偏航（左右转头），就让1、3转快点、2、4转慢点，产生净反力矩。

反力矩作用在机体上：

$$\tau_g = -J_r \Omega_{\text{net}} (\omega \times e_3)$$

**答辩话术**："转动动力学采用完整的欧拉方程，包含陀螺耦合项和旋翼反扭矩陀螺项。后者是四旋翼特有的动力学特性：当相邻旋翼产生净转速差时，高速旋转的旋翼会对机体产生陀螺反力矩，直接影响滚转和俯仰通道。"

### 2.4 姿态运动学

#### 欧拉角形式

知道当前角度和角速度，用这组方程算下一秒角度：

$$\dot \phi = p + q\sin\phi\tan\theta + r\cos\phi\tan\theta$$
$$\dot \theta = q\cos\phi - r\sin\phi$$
$$\dot \psi = q\frac{\sin\phi}{\cos\theta} + r\frac{\cos\phi}{\cos\theta}$$

危险：当 $\theta = 90°$ 时，$\cos\theta = 0$，方程会**除零**！这就是"万向节锁死"。

#### 四元数形式（更安全）

四元数用4个数表示旋转，没有万向节锁死：

$$\dot q = \frac{1}{2} \Omega(\omega) q$$

$q = [q_w, q_x, q_y, q_z]$ 是四元数，$\Omega(\omega)$ 是由角速度构造的4×4矩阵。

**数值积分**：每隔0.01秒算一次"现在在哪"。文档用**RK4（四阶龙格-库塔法）**，比简单的"速度×时间"精确得多。四元数积分后要**归一化**（缩放回长度1）。

**答辩话术**："姿态运动学同时提供欧拉角和四元数两种描述。数值积分采用四阶龙格-库塔方法保证精度，四元数积分后执行单位化以维持旋转表示的数学约束，避免万向节锁死问题。"

### 2.5 控制分配：从"意图"到"四个螺旋桨"

你心里想的是"我要上升""我要往前飞"，但飞机只懂"四个螺旋桨各自转多快"。需要"翻译器"：把4个高层指令翻译成4个电机转速。

4个高层指令：
- $T$：总推力（管上升/下降）
- $M_x$：绕x轴力矩（管俯仰）
- $M_y$：绕y轴力矩（管滚转）
- $M_z$：绕z轴力矩（管偏航）

4个电机推力 $f_1, f_2, f_3, f_4$ 的关系：

$$\begin{bmatrix}T\\M_x\\M_y\\M_z\end{bmatrix} = \underbrace{\begin{bmatrix}1&1&1&1\\0&L&0&-L\\-L&0&L&0\\k_m/k_f&-k_m/k_f&k_m/k_f&-k_m/k_f\end{bmatrix}}_{\text{控制分配矩阵 }W} \begin{bmatrix}f_1\\f_2\\f_3\\f_4\end{bmatrix}$$

矩阵怎么读：
- 第一行：总推力 = 四个旋翼之和
- 第二行：绕x轴力矩 = 旋翼2的力矩（力臂L）减旋翼4的力矩
- 第三行：绕y轴力矩 = 旋翼3的力矩减旋翼1的力矩
- 第四行：绕z轴力矩 = 旋翼反扭矩之和（系数 $k_m/k_f$ 是力矩/推力比）

实际代码做**逆问题**：已知想要的 $T, M_x, M_y, M_z$，反求四个 $f_i$。通过矩阵求逆或伪逆实现。

**答辩话术**："控制分配矩阵建立了期望推力/力矩与四个旋翼推力之间的线性映射。矩阵的第一行对应总推力，第二、三行分别对应俯仰和滚转力矩（力臂为L），第四行编码旋翼反扭矩产生的偏航力矩。工程实现中通过矩阵求逆完成从高层控制指令到电机底层推力的解算。"

## 3. 控制器设计

### 3.0 什么是"控制器"？

你开车目标是"保持在车道中央"。你眼睛看路面（**测量**），发现车偏右了（**误差**），你向左打方向盘（**控制动作**），车回正了。控制器就是干这个的。

### 3.1 串级PID（最主流的工业控制器）

PID = Proportional（比例）+ Integral（积分）+ Derivative（微分）

- **P（比例）**：误差越大，动作越大。偏右1米打1圈方向盘，偏右2米打2圈
- **I（积分）**：如果一直有个小误差没消除（车总偏右5厘米），积分慢慢积累，最终磨掉
- **D（微分）**：预测未来。车正在快速冲向右边，即使现在还在中央，也要提前打方向

#### 四旋翼的"四层套娃"

四旋翼控制是**一层套一层**的：

```
最外层：位置环 —— "我要飞到(10,20,5)"
    ↓ 输出：期望速度
中间层：速度环 —— "我需要以2m/s的速度飞"
    ↓ 输出：期望加速度
内层：姿态环 —— "我需要倾斜15°来产生这个加速度"
    ↓ 输出：期望角速度
最内层：角速度环 —— "我需要以1rad/s的速度改变倾斜"
    ↓ 输出：期望力矩
```

**为什么分层？** 位置变得慢（从A飞到B要几秒），速度变中等，姿态变快，角速度变得最快（旋翼响应毫秒级）。混在一起控制会"打架"，分层后外环给内环下命令，内环只管快速执行。

#### 位置环

$$v_{\text{des}} = v_d + K_{p,p} \cdot e_p + K_{i,p} \int e_p \, dt$$

- $v_d$：目标速度
- $e_p = p_{\text{target}} - p_{\text{current}}$：位置误差
- $K_{p,p}$：位置比例增益
- $K_{i,p}$：位置积分增益（慢慢消除稳态误差）

#### 速度环

$$a_{\text{cmd}} = a_d + K_{p,v} \cdot e_v + K_{i,v} \int e_v \, dt + K_{d,v} \frac{e_v(k) - e_v(k-1)}{\Delta t}$$

多了**微分项**（D项）：$\frac{e_v(k) - e_v(k-1)}{\Delta t}$ 是速度误差变化率，预测飞机是不是"冲过头"了。

#### 姿态环

$$\omega_d = \text{clip}\left(K_{p,\eta} \cdot e_\eta,\ -\omega_{\max},\ \omega_{\max}\right)$$

$\eta$ 代表姿态角（$\phi, \theta, \psi$）。姿态环**只加比例**，并且**限幅**（不能超过最大角速度），防止翻转过快失控。

#### 小角度近似：从加速度反解姿态

想往右飞，不能直接"往右推"，只能**倾斜机身**，让推力的水平分量带着你走。

小角度下：

$$\phi_d = \frac{a_x \sin\psi_d - a_y \cos\psi_d}{g}$$

需要的水平加速度越大，倾斜角度越大。$g$ 是重力加速度。

#### 角速度环

$$\tau = I \left( K_{p,\omega} e_\omega + K_{i,\omega} \int e_\omega \, dt + K_{d,\omega} \frac{e_\omega(k) - e_\omega(k-1)}{\Delta t} \right)$$

输出是**力矩**，送入前面的控制分配矩阵，转成四个旋翼的推力。

#### 抗饱和限幅

你一直推方向盘到底，但车已经转不过来了。积分器还在积累"我要更多转弯"的误差，等你松开方向盘，车会过度反应。这就是**积分饱和**。

解决办法：给积分项加**上限**。超过上限就不再加，防止"累积的怒气值"爆表。

**答辩话术**："串级PID采用四层嵌套结构：位置外环输出期望速度，速度环输出期望加速度，姿态环将加速度指令解算为期望滚转/俯仰角，角速度最内环跟踪期望角速度并输出控制力矩。每层均可独立配置PID增益，积分项全部带抗饱和限幅。小角度近似下，水平加速度需求通过重力加速度归一化反解期望姿态角，这是四旋翼飞行控制的核心机理。"

### 3.2 反步法控制器（Backstepping）

#### 直觉

学骑自行车：
1. 第一层：你想去某个地方（位置目标）
2. 第二层：你需要以某速度到达那里（速度目标）
3. 第三层：你需要倾斜车身来产生速度（姿态目标）

反步法就是**一步一步往回推**：先设计最外层理想行为，然后证明"如果内层能做到xxx，外层就稳定"，再设计内层控制器让"xxx"成立。

#### Lyapunov函数（稳定性证书）

数学家Lyapunov的方法：找到"能量函数" $V$，满足 $V \ge 0$、$\dot V \le 0$、$V=0$ 只在平衡点，那么系统一定**收敛**。

文档里构造：

$$V = \frac{1}{2} z_0^T K_0 z_0 + \frac{1}{2} z_1^T z_1 + \frac{1}{2} z_2^T z_2$$

- $z_0$：位置误差的积分
- $z_1$：位置误差
- $z_2$：速度误差

求导：

$$\dot V = -z_1^T K_1 z_1 - z_2^T K_2 z_2 \le 0$$

证明了系统是**稳定的**。

**答辩话术**："反步法通过递归构造Lyapunov函数证明闭环稳定性。定义积分位置误差、位置误差和速度误差三层虚拟控制量，构造正定Lyapunov函数并证明其沿系统轨迹的导数半负定，从而保证误差状态渐近收敛。工程实现中引入饱和限幅，严格理论保证退化为实用稳定性。"

### 3.3 滑模控制（SMC）

#### 直觉

你在冰面上走路。滑模控制思想：不管地面怎么滑，你强行沿着一条"滑道"走。只要在滑道上，系统就是稳定的。

#### 滑模面

定义"理想轨道"：

$$s = e_v + \Lambda e_p$$

$s=0$ 意味着 $e_v = -\Lambda e_p$，即"如果你离目标还很远，就飞快点；快到了，就慢下来"。

#### 控制律

$$u_{\text{sw}} = \Lambda e_\omega + \epsilon \, \text{sat}\left(\frac{s}{\delta}\right) + K_s s + K_i \int s \, dt$$

- $\epsilon \, \text{sat}(s/\delta)$：滑模切换项。$\epsilon$ 要大于扰动上界。sat是饱和函数（平滑过渡），减少抖振
- $K_s s$：比例反馈，把系统拉回滑模面
- $K_i \int s \, dt$：积分项，消除稳态误差

**为什么用sat而不是sign？**

sign函数在 $s=0$ 附近会高频切换（chattering，抖振），像快速按开关，系统会抖。sat函数在边界层 $\delta$ 内平滑过渡。

**答辩话术**："滑模控制设计线性滑模面 $s = e_v + \Lambda e_p$，将二阶误差动力学约束到滑模流形上。控制律包含滑模切换项、比例反馈项和积分补偿项。采用饱和函数替代符号函数，在边界层厚度 $\delta$ 内实现平滑过渡，有效抑制传统滑模控制的抖振现象，同时保证到达条件 $\epsilon > \bar d$ 满足。"

## 4. 风场模型

### 4.1 什么是风？

风就是**空气在流动**。飞机在风里飞，感受到的是**相对于风的速度**。

### 4.2 模型

```
稳态风：固定方向和大小（比如一直吹东风5m/s）
    +
湍流风：随机扰动（像一阵阵风）
```

湍流风用**Ornstein-Uhlenbeck过程**模拟：

$$w_k = \alpha w_{k-1} + (1-\alpha) \epsilon_k$$

直觉：当前风速 = 上一时刻风速的 $\alpha$ 倍 + 随机扰动。$\alpha$ 接近1表示风变化慢，接近0表示风变化快。

风进入阻力项 $F_d$，影响飞机运动，用来**测试控制器抗干扰能力**。

**答辩话术**："风场模型包含稳态风和湍流风两种成分。湍流采用Ornstein-Uhlenbeck一阶滤波生成，通过调节惯性系数 $\alpha$ 控制风速的时间相关性。风扰动通过非线性阻力项耦合进入平动动力学，用于验证控制器的鲁棒性。"

## 5. 编队拓扑基础

### 5.1 编队偏移：每架飞机站在哪？

假设你是领队（leader），其他飞机跟着你，每架有自己的"站队位置"。

```
       从机2
        ↑
从机1 ← L(领队) → 从机3
        ↓
       从机4
```

每架从机（follower）的目标位置：

$$p_i^d = p_L + R_z(\psi_L) o_i$$

拆解：
- $p_L$：领队位置
- $\psi_L$：领队朝向
- $o_i$：从机i的相对偏移（"在我左后方3米"）
- $R_z(\psi_L)$：把相对偏移旋转到领队的朝向方向

### 5.2 支持的队形

| 队形 | 形状 | 应用场景 |
|------|------|---------|
| V形 | 像大雁 | 搜索、飞行表演 |
| 菱形 | 前后左右各一架 | 防御阵型 |
| 线形 | 一字排开 | 农田喷洒 |
| 三角形 | 三角分布 | 摄影补光 |

### 5.3 拓扑连通度

编队不仅要看形状，还要看**通信**。用**图**表示通信关系：
- 节点 = 飞机
- 边 = 两架飞机能互相通信

**拉普拉斯矩阵**：

$$L = D - A$$

- $D$：度矩阵（对角线上是每个飞机的连接数）
- $A$：邻接矩阵（谁和谁连着）

**代数连通度** $\lambda_2$（第二小的特征值）：
- $\lambda_2 > 0$：所有人能间接联系到彼此（连通）
- $\lambda_2$ 越大：通信网络越健壮

**论文 [4] Lan et al. (IEEE TAES 2025)**：故障后拓扑重构时选连通度最高的队形。

**答辩话术**："编队采用领航-跟随者架构，从机目标位置由领队位姿经旋转矩阵映射得到，支持V形、菱形、线形、三角形及自定义队形。拓扑连通度采用图拉普拉斯矩阵的第二小特征值 $\lambda_2$ 度量，该值大于零当且仅当通信图连通，值越大代表编队对通信链路失效的鲁棒性越强。"

## 6. 障碍物与地图

### 6.1 障碍物原语

代码里定义了几种基本形状：

| 形状 | 参数 | 例子 |
|------|------|------|
| AABB（轴对齐长方体） | 中心 + 半长宽 | 房间墙壁 |
| Sphere（球） | 中心 + 半径 | 气球、树 |
| Cylinder（圆柱） | 中心 + 半径 + 高 | 柱子 |

### 6.2 SDF（有符号距离场）

想象你在房间里，每个位置标一个数字：
- **正数**：离最近的墙多远（在房间内部）
- **负数**：在墙里面多深（穿墙了）
- **0**：正好在墙面上

这就是**有符号距离**（Signed Distance Field, SDF）。

#### AABB的SDF

$$d_{\text{box}}(p) = \|\max(q, 0)\|_2 + \min(\max(q_x, q_y, q_z), 0)$$

- 第一项：如果你在盒子外面，算你到最近表面的距离
- 第二项：如果你在盒子里面，给你负的距离
- 合起来：正负号表示内外，大小表示远近

#### Sphere的SDF

简单：$d = \|p - c\|_2 - r$

### 6.3 占据栅格与ESDF

**占据栅格**：把3D空间切成小方块（体素），标记1=有障碍，0=空的。

**ESDF（欧氏有符号距离场）**：对每个空体素，算它到**最近的占据体素**的距离。

**软代价**：

$$c_{\text{extra}}(i) = w \exp\left(-\frac{d_{\text{esdf}}(i)}{s}\right)$$

- 离障碍物很近：代价 $\approx w$（很贵）
- 离障碍物很远：代价 $\approx 0$（免费）

路径总代价 = 路径长度 + $\lambda \times$ 软代价和。规划器会**主动远离障碍物**。

**答辩话术**："障碍物几何采用AABB、球体和圆柱三种基本原语，通过有符号距离场实现统一的几何查询接口。SDF的符号表示点在障碍物内外，绝对值表示到最近表面的距离，梯度指向自由空间方向。ESDF在占据栅格上全局计算欧氏距离，并构造指数软代价函数——距离障碍物越近代价越高，引导规划器选择不仅无碰撞而且保持安全裕度的路径。"

## 7. 路径规划

### 7.0 为什么要规划？

你在迷宫里走路，需要**提前想好走哪条路**才能不撞墙。无人机也一样：在3D空间里飞，障碍物是墙、柱子、其他飞机，需要算法**自动找路**。

### 7.1 统一接口

所有规划器都实现同样的接口：

```python
plan(start, goal, grid) -> path
smooth(path) -> smoothed_path
```

就像所有手机充电口形状一样，可以随便换。

### 7.2 A*算法

#### 直觉

你在格子地图上找路。每个格子有代价（走1米算1分）。

A* 维护两个值：
- $g(n)$：从起点到当前格子 $n$ 的实际代价
- $h(n)$：从 $n$ 到终点的**估计代价**（启发函数）

总评分：

$$f(n) = g(n) + h(n)$$

每次都选 $f$ 最小的格子扩展。

#### 启发函数

文档用**欧氏距离**：

$$h(n) = \|p_n - p_g\|_2$$

这是"直线距离"，满足**可采纳性**：永远不会高估实际代价。可采纳性保证A*一定能找到最优路径。

#### ESDF增强

加入软代价后，实际代价变成：

$$\text{实际代价} = \text{路径长度} + \lambda \times \sum \text{软代价}$$

A*不仅找最短的，还找**最安全的**。

**答辩话术**："A*采用欧氏启发函数，对3D网格路径长度可采纳，保证最优性。通过引入ESDF软代价，优化目标从单纯最短路径扩展为路径长度与安全裕度的加权组合，使规划结果既短且远离障碍物。"

### 7.3 Hybrid A*

#### 为什么需要它？

标准A*假设你可以**瞬移到相邻格子**。但真实飞机不能瞬移，它有朝向，转弯需要时间。

Hybrid A* 是Google无人车（Stanford的Junior）用的算法，论文 **[7] Dolgov et al., AAAI 2008**。

#### 核心思想

状态不是只有位置 $(x,y,z)$，而是加上**朝向角 $\psi$**：

$$x = (p_x, p_y, p_z, \psi)$$

飞机沿着**运动原语**（motion primitives）积分一段轨迹，而不是"跳到"相邻格子。

#### 13个离散动作

前进、左移、右移、上升、下降，加上左转前进、右转前进等，共13种基本动作。每个动作都按照飞机的实际运动学积分一小段时间。

#### 启发函数

两个启发函数取**最大值**（保证可采纳）：
1. 反向Dijkstra：从终点反向搜索，考虑障碍物
2. 无障碍下界：忽略障碍物，按最小转弯半径的Dubins曲线算最短时间

搜索时状态用**离散化索引**：位置栅格化 + 朝向分成72个桶（每5°一个桶）。

**答辩话术**："Hybrid A*引入连续姿态维度，状态扩展通过13个运动原语按真实运动学积分实现，而非简单的格子跳跃。启发函数组合反向Dijkstra避障代价与无障碍最小时间下界，两者取最大值保证可采纳性。状态空间采用位置栅格索引加72桶离散航向的混合离散化，是计算效率与路径连续性的平衡。"

### 7.4 RRT* 与 Informed RRT*

#### RRT（快速探索随机树）

想象你在黑暗中探索洞穴：
1. 从起点扔一根绳子
2. 随机选一个方向伸出一小段
3. 如果那段没撞墙，就固定住，继续从那里伸
4. 直到碰到终点附近

RRT通过**随机采样**快速探索空间。

#### RRT* 的改进（论文 [8] Karaman & Frazzoli, IJRR 2011）

原始RRT有个问题：它**几乎必然不是最优的**——树一旦长成就不管了，不会重新优化。

RRT* 加了两个关键操作：
1. **选择父节点**：新节点不只连到最近点，而是连到能让总代价最小的邻居
2. **重连（Rewiring）**：检查已有节点能否通过新节点获得更短路径，如果可以，就"改嫁"

**渐近最优性**：随着采样点越来越多，RRT* 的路径长度**几乎必然收敛到最优**。

#### 重连半径

$$r_n = \min\left(r_{\max},\ \gamma \left(\frac{\log(n+1)}{n+1}\right)^{1/d}\right)$$

- $n$：采样点数
- $d$：维度（3D空间里 $d=3$）
- $\log(n)/n$ 越来越小：采样多了，重连半径变小，保证计算量可控

#### Informed RRT*（论文 [9] Gammell et al., IROS 2014）

RRT* 找到第一条路径后，Informed RRT* 说："不用再在全空间乱采样了，最优路径一定在某个范围内！"

这个范围是一个**椭球**：
- 两个焦点：起点和终点
- 横轴长度：当前最优路径长度
- 只有在这个椭球内的点才值得采样

直接在这个椭球内采样，效率比在全空间采样**高一个数量级**。

**答辩话术**："RRT*通过近邻重连机制实现渐近最优性，重连半径按对数收缩率 $r_n \propto (\log n/n)^{1/d}$ 设计，平衡计算复杂度与理论收敛保证。Informed RRT*在获得初始解后，将采样空间收缩到以起点和终点为焦点、当前最优路径长度为横轴的椭球子集，直接在该子集中采样避免在高维空间的无谓探索，收敛速度提升一个数量级。"

### 7.5 D* Lite

#### 问题背景

你在走路，地图在变化（突然有人搬来一堆箱子）。A* 和 RRT* 遇到这种情况需要**重新算一遍**。

D* Lite 说："地图只变了局部，我只需要更新受影响的部分。"

#### 核心概念

维护两个值：
- $g(s)$：起点到当前节点的估计代价（类似A*的g值）
- $rhs(s) = \min_{s'\in\text{后继}}(c(s,s') + g(s'))$：一步前瞻值

**一致性**：
- $g(s) = rhs(s)$：节点"一致"，信息是最新的
- $g(s) > rhs(s)$：节点"过度估计"，需要降低
- $g(s) < rhs(s)$：节点"不足估计"，需要升高

#### 增量更新

当障碍物变化时：
1. 只把直接受影响的边加入优先队列
2. 传播不一致性，直到所有相关节点重新一致
3. 机器人移动时，通过键值 $k(s)$ 中的 $k_m$ 项修正启发值，避免重排队列

**论文 [10] Koenig & Likhachev, AAAI 2002**

**答辩话术**："D* Lite是增量式启发搜索算法。维护 $g$ 值和前瞻 $rhs$ 值，通过局部一致性检验实现障碍变化的增量更新。优先队列键值包含启发修正项 $k_m$，解决机器人移动后启发值过时问题。相比全局重规划，D* Lite在局部环境变化时只更新受影响子图，计算效率提升显著。"

### 7.6 路径平滑（Catmull-Rom样条）

规划出的路径是折线，飞机沿着折线飞会抖。Catmull-Rom样条是**插值方法**：给定几个控制点，生成一条光滑曲线，保证**一阶连续**（位置连续、切线方向连续）。

平滑后还需要把曲线采样成密集点，再做SDF校核，保证安全。

**答辩话术**："路径后处理采用Catmull-Rom样条实现 $C^1$ 连续平滑，消除折线轨迹的执行抖动。平滑后执行线段采样与SDF校核的二次安全验证，确保几何平滑不牺牲碰撞安全性。"

## 8. FIRI安全走廊

### 8.0 什么是安全走廊？

你在长满荆棘的小路上走。安全走廊就是：**把荆棘推远，给你修一条两边有护栏的通道**，你在护栏中间走就一定安全。

对无人机来说，安全走廊是一个**凸多面体区域**（像被很多平面围成的盒子），保证区域内没有障碍物。

### 8.1 半空间表示

一个凸多面体可以用一堆不等式表示：

$$\mathcal C = \{x \mid Ax \le b\}$$

每个不等式 $a_i^T x \le b_i$ 表示一个**半空间**（平面的一侧）。交集就是凸多面体。

### 8.2 解析分离平面（怎么造护栏）

对seed线段附近每个障碍物，找离线段最近的点，构造一个平面把障碍物隔在外面：

平面法向：

$$n = \frac{q - p_o}{\|q - p_o\| + \epsilon}$$

- $q$：线段上离障碍物最近的点
- $p_o$：障碍物表面上离 $q$ 最近的点
- $n$：从障碍物指向 $q$ 的方向（护栏朝向）

多个障碍物时，每个障碍物都贡献一个护栏，最终走廊是所有护栏的交集。

**凹角问题**：如果两个障碍物形成很窄的凹角，凸走廊没法精确表示。此时退回SDF推离。

### 8.3 MVIE（最大体积内接椭球）

想让走廊尽可能"宽敞"，在里面放一个最大的椭球，椭球不碰任何障碍物。

数学优化：

$$\max_{C,d} \log\det C$$
$$\text{s.t. } \|Ca_i\|_2 + a_i^T d \le b_i$$

- $C, d$：椭球的形状和中心
- 约束：椭球完全在所有半空间内

用 `cvxpy` + `CLARABEL/SCS` 求解。如果凸优化失败，退回内接球近似。

### 8.4 航点投影与SDF推离

把路径上的点投影到走廊内。如果投影后离障碍物还是太近，就沿SDF梯度方向推远：

$$p_{\text{new}} = p + (r_{\text{safe}} - d_{\text{field}}(p)) \nabla d_{\text{field}}(p)$$

**答辩话术**："FIRI安全走廊通过解析分离平面构造凸多面体半空间表示，每个障碍物独立贡献分离平面，最终走廊为所有有效半空间与seed轴向包围盒的交集。遇到凹角空间时以保守凸子集近似，若过窄则退回SDF推离。通过求解MVIE凸优化问题最大化走廊内接椭球体积，失败时以内接球近似退避。航点投影后执行SDF推离二次安全强化。"

## 9. 避碰系统

### 9.1 人工势场（APF）

#### 直觉

终点对你有**吸引力**，障碍物对你有**排斥力**。合力就是你该走的方向。

#### 公式

吸引力：

$$U_{\text{att}} = \frac{1}{2} k_{\text{att}} \|p - p_{\text{goal}}\|^2$$

排斥力：

$$U_{\text{rep}} = \frac{1}{2} k_{\text{rep}} \left(\frac{1}{d} - \frac{1}{r_{\text{rep}}}\right)^2 \|p - p_{\text{goal}}\|^n$$

- $d$：到障碍物距离
- $r_{\text{rep}}$：排斥半径（超过这个距离就不排斥了）
- $n$：增益调节（远处弱、近处强）

#### APF的经典问题

1. **局部最小值**：吸引力=排斥力时，合力为零，飞机停住不动
2. **振荡**：在障碍物和目标之间来回弹

#### 改进：旋转势场（论文 [1] Pan et al., IEEE TCSII 2022）

Pan等人提出的IAPF：当检测到 $\|F\| \le \eta$（陷入局部最小）时，加一个**旋转势场** $U_e$：

$$U_e = \frac{1}{2} k_e \alpha^2 \sum_{j=1}^{N} d_{j,\text{min}}$$

产生一个垂直于引力方向的切向力，让飞机"绕个圈"脱困。

**答辩话术**："人工势场作为执行层局部避障加速度生成器，而非全局规划替代。斥力势场采用距离倒数的平方形式，在排斥半径外自动归零。系统引入Pan等人提出的旋转势场改进机制：当合力模长低于阈值 $\eta$ 判定为局部最小值时，旋转势场提供垂直于引力方向的切向分量，引导无人机绕过局部陷阱，有效缓解传统APF的振荡和死锁问题。"

### 9.2 多层安全链路

系统设计了**8层防护**：

```
第1层：全局路径规划（A*/RRT*）     —— 宏观路线
第2层：SDF路径校核                  —— 路径本身安全
第3层：线段采样加密                 —— 检查路径上的每个小段
第4层：在线前视目标安全裁剪          —— 下一步目标点安全
第5层：从机安全目标投影             —— 编队成员不撞
第6层：APF实时排斥                  —— 动态障碍物推开
第7层：执行层SDF安全屏障            —— 硬约束兜底
第8层：控制器稳定跟踪               —— 精确执行
```

**阈值分离**：
- $r_{\text{plan}}$：规划时用的安全距离（大，留余量）
- $r_{\text{detect}}$：执行时用的检测距离（小，更严格）

**答辩话术**："系统构建八层递进式安全链路，从宏观规划到微观执行层层设防。规划阈值与检测阈值分离设计：规划半径包含编队包络和安全裕度，检测半径为机体半径加缩放的裕度，实现规划保守性与执行敏捷性的平衡。"

### 9.3 从机安全目标投影

编队过窄通道时，从机的编队偏移位置可能撞墙。解决：把从机位置沿**到领队的方向**往回收缩：

$$p_i(\lambda) = p_L + \lambda \cdot \Delta_i$$

找最大的 $\lambda$ 使得：

$$\text{SDF}(p_i(\lambda)) \ge r_{\text{follower}}$$

**答辩话术**："窄通道场景下执行从机安全目标投影：沿从机-领队偏移向量线性缩放，收缩编队间距直至满足跟随机安全半径，实现队形保持与避障的协调。"

### 9.4 执行层SDF安全屏障

每步积分后检查：如果飞机位置到障碍物的距离小于检测半径，**强制修正**：

1. 把位置沿SDF梯度推远到安全距离
2. 清除速度在障碍物法向方向的分量（防止"速度穿墙"）

**答辩话术**："执行层SDF安全屏障作为最终硬约束兜底：每步积分后若距离低于检测阈值，执行位置投影和速度法向分量清除，保证物理轨迹始终满足 $\text{SDF}(p(t)) \ge r_{\text{detect}}$。该机制不替代规划器，但防止动力学惯性导致的走廊穿越。"

## 10. GNN双模式规划

### 10.1 Safe/Danger调度

飞机根据周围环境切换两种模式：

| 模式 | 触发条件 | 行为 |
|------|---------|------|
| Safe | 周围很空 | 正常用A*/Hybrid A*/RRT*规划 |
| Danger | 离障碍物很近 | 切换到GNN局部绕障 |

**滞回切换**：
- 进Danger：SDF危险 **或** 传感器危险（容易触发）
- 回Safe：SDF安全 **且** 传感器安全（严格确认）

### 10.2 可见图（Visibility Graph）

在障碍物顶点之间建图，如果两点之间**直线看得见**（线段不碰障碍物），就连一条边。

顶点采样：
- AABB：8个角点
- 圆柱：顶圆和底圆采样
- 球：Fibonacci球面均匀采样

边验证：线段SDF校核，保证全程安全。

### 10.3 GNN活动扩散（神经动力学）

**重要澄清**：这个GNN**不是**深度学习里的图神经网络！没有训练过程，没有数据集。

它是**生物神经动力学**的数学模型（Grossberg, 1980s），用于解决路径规划。

#### 活动值更新方程

$$\frac{dx_i}{dt} = -Ax_i + (B-x_i)\left(I_i + \sum_j w_{ij}x_j\right) - (D+x_i)J_i$$

拆解：
- $-Ax_i$：自然衰减（神经元会疲劳）
- $(B-x_i)(\dots)$：兴奋项（邻居激活你，但上限是 $B$）
- $-(D+x_i)J_i$：抑制项（障碍物抑制你）

#### 激励项

$$I_i = V \cdot \frac{1}{(1+d_{i,\text{goal}})^\alpha} \cdot \frac{1}{(1+d_{i,\text{path}})^\beta}$$

- 离终点越近，激励越大
- 离全局参考路径越近，激励越大

#### 连接权重

$$w_{ij} = \frac{1}{(1+d_{ij})^\gamma}$$

两点越近，连接越强（局部连接性）。

#### 路径提取

等活动扩散**收敛**（稳定）后：
1. 从起点开始
2. 找邻居中**活动值最大**的
3. 保证可见性（直接连边）
4. 贪心走到终点

如果贪心卡住，用BFS找未访问的续接点。

**论文对应**：[3] Yuan et al., 2020（多信息膨胀图）；[6] Wang et al., 2007（合作UAV编队飞行的GNN应用）。

**答辩话术**："Danger模式采用可见图与GNN神经动力学活动扩散相结合的局部重规划。可见图从障碍物几何采样顶点并执行边可见性校核，将搜索空间从全栅格压缩到障碍物顶点集。GNN活动扩散基于Grossberg神经动力学方程，激励项由终点接近度和全局路径偏离度加权构成，连接权重与顶点间距负相关。收敛后沿活动值梯度贪心提取路径，若局部极值导致死锁则回退BFS续接。该机制Danger模式下不依赖全局重规划，响应速度快。"

### 10.4 Danger路径质量闭环

GNN路径必须通过两个检查：
1. **可见性校核**：路径上所有点连线安全
2. **长度比校核**：$L_{\text{gnn}} / L_{\text{local}} \le$ 阈值

否则回退到局部A*规划。

**答辩话术**："GNN路径提取后执行质量闭环：可见性校核保证几何安全，长度比校核防止GNN路径过度迂回。任一检查失败即回退到滑动窗口A*局部规划，保证Danger模式的路径质量可控。"

## 11. 故障检测与拓扑重构

### 11.1 故障注入

模拟旋翼损坏：$f_i^{\text{actual}} = \mu_i f_i$，$\mu_i \in [0,1]$

- $\mu_i = 1$：健康
- $\mu_i = 0.5$：推力减半（部分失效）
- $\mu_i = 0$：完全失效（旋翼停转）

### 11.2 故障检测（论文 [4] Lan et al., IEEE TAES 2025）

三个判据，满足**任一**且持续一定时间就报警：

| 判据 | 含义 | 持续时间 |
|------|------|---------|
| $F_{\text{acc}}$ | 加速度异常 | 0.5秒 |
| $F_{\text{pos}}$ | 位置偏差大 | 1.0秒 |
| $F_{\text{sat}}$ | 执行器饱和 | 连续多步 |

**为什么用"或"逻辑？** 故障表现形式多样：有的反映在动力学异常（加速度），有的反映在跟踪失败（位置），有的反映在硬件极限（饱和）。三者互补，降低漏检率。

**论文 [4]**：采用FDI（Fault Detection and Isolation）方法检测执行器故障和拓扑故障。

**答辩话术**："故障检测采用多判据或逻辑融合策略，包含加速度异常、位置跟踪偏差和执行器饱和三项独立指标，持续时间阈值分别为0.5秒和1.0秒。或逻辑设计覆盖不同故障表现形式：动力学突变反映在加速度判据，控制器失效反映在位置判据，硬件极限反映在执行器饱和判据，降低单一检测机制的漏报风险。"

### 11.3 拓扑重构

假设原来5架飞机排成V形，1架坏了。剩下4架怎么办？

```
Step 1: 检测到故障
        ↓
Step 2: 确定健康飞机数 N_h（原来5架，坏了1架，N_h=4）
        ↓
Step 3: 按 N_h 生成候选队形
        - V形4机版、菱形4机版、线形4机版...
        ↓
Step 4: 对每个候选队形，算通信图和 λ₂
        - 建邻接矩阵（基于预设通信范围）
        - 算拉普拉斯矩阵 L
        - 算 λ₂
        ↓
Step 5: 选 λ₂ 最大的队形，执行平滑切换
```

**为什么选最大的？** $\lambda_2$ 越大，队形对通信链路断开的容忍度越高。

**论文 [4] 的核心贡献**：Lan 等人提出**最小能量约束下的拓扑重构**。本系统借鉴其"按健康飞机数重新生成队形并选连通性最强者"的思想，同时保证切换轨迹的平滑性。

### 11.4 候选队形生成规则

| $N_h$ | 候选队形 |
|-------|---------|
| 2 | 线形（前后） |
| 3 | 三角形、V形 |
| 4 | 菱形、V形、线形 |
| 5 | V形、菱形、三角形、线形 |

**关键**：不是简单"删掉坏飞机、其他人不动"。例如原来5架V形，中间领队+左右各两架。如果左翼最外侧坏了，剩下4架如果直接删掉那个位置，队形就偏了。系统会**重新生成对称的4架队形**（比如菱形或紧凑V形）。

**答辩话术**："拓扑重构不是简单地删除故障节点保留其余偏移量，而是按健康无人机数 $N_h$ 从队形库中重新生成对称候选队形。例如五机V形故障后，不是保留不对称四机分布，而是切换至四机菱形或紧凑V形，保证编队几何对称性与重心稳定性。"

## 12. 实时规划（在线重规划）

### 12.1 为什么需要实时规划？

**离线规划**（标准A*、RRT*）有个大前提：**地图不能变**。但真实世界里：
- 地图有误差（墙上多了个箱子）
- 有动态障碍物（突然走进来一个人）
- 传感器实时发现了新障碍

如果地图变了，离线规划器**完全不知道**，飞机就照着旧路径撞上去了。

**实时规划**：飞机在飞的过程中，每隔一小段时间（0.01秒），都要**重新看一遍周围**，**快速决定下一步往哪走**。

### 12.2 实时规划的难点

| 难点 | 为什么难 |
|------|---------|
| **算力不够** | 地图很大，全局重算太慢（0.01秒根本算不完） |
| **信息不全** | 只能看到周围一小片，远处不知道 |
| **不能断链** | 每一步都要有完整到终点的路径 |
| **要平滑** | 不能这一步往东、下一步往西 |
| **要安全** | 每一步都要保证不撞 |

### 12.3 三级分层架构

```
┌─────────────────────────────────────────────────┐
│  第一层：全局参考路径（Global Reference Path）      │
│  工具：Informed RRT*                              │
│  频率：启动时算一次                                  │
│  范围：从起点到终点的完整粗略路径                     │
│  作用：告诉系统"大致往哪个方向走"                    │
└──────────────────────┬──────────────────────────┘
                       ↓ 提供参考方向和大致路线
┌─────────────────────────────────────────────────┐
│  第二层：增量更新（D* Lite）                       │
│  频率：地图局部变化时触发                            │
│  范围：变化区域及其影响范围                          │
│  作用：局部修路径，不用全局重算                      │
└──────────────────────┬──────────────────────────┘
                       ↓ 提供修好的参考路径
┌─────────────────────────────────────────────────┐
│  第三层：局部滑动窗口（Sliding Window A*）         │
│  频率：每步都跑（最高频）                            │
│  范围：以飞机为中心，周围20米的立方体                 │
│  作用：算精细的局部路径                              │
└─────────────────────────────────────────────────┘
```

**类比**：开车去陌生城市：
- **全局层**：出发前看高德地图，知道"先上高速，再下X出口"
- **增量层**：高速上发现某段堵车，高德帮你绕一下
- **局部层**：每到路口，看指示牌决定左转还是右转

### 12.4 局部滑动窗口

#### 什么是"窗口"？

飞机周围有一个**透明的立方体盒子**：

```
            ┌─────────────┐
           ╱             ╱│
          ╱   窗口中心   ╱ │
         ╱    (飞机)    ╱  │  ← 窗口边长 = window_size
        ├─────────────┤   │    (比如 20米)
        │             │   │
        │             │   │
        │      ✈      │   │
        │    (飞机)   │  ╱
        │             │ ╱
        │             │╱
        └─────────────┘
```

**窗口特点**：
- 中心跟着飞机移动（"滑动"窗口）
- 只处理窗口内的障碍物（窗外不管）
- 只在窗口内做A*搜索（搜索空间小，算得快）

#### 窗口栅格化

窗口大小20米，体素0.5米：
- 每边 20/0.5 = 40 个格子
- 总共 40×40×40 = 64,000 个格子

A*在这个小网格上搜索，很快（毫秒级）。

#### 窗口A*的代价函数

$$f(n) = g(n) + h(n) + \lambda_{\text{esdf}} \cdot c_{\text{esdf}}(n) + \lambda_{\text{shortcut}} \cdot c_{\text{shortcut}}(n)$$

| 项 | 含义 |
|----|------|
| $g(n)$ | 从当前位置到n的实际代价 |
| $h(n)$ | 从n到子目标的启发估计（欧氏距离） |
| $c_{\text{esdf}}(n)$ | ESDF软代价（离障碍物越近越贵） |
| $c_{\text{shortcut}}(n)$ | 捷径代价（鼓励走直线） |

**为什么加ESDF代价？** 不只找最短的，还找**最安全的**——走大道不走小巷。

### 12.5 子目标选择（最关键的难题）

#### 为什么要选子目标？

窗口只有20米，但终点可能在**1000米外**。局部A*不能直接以终点为目标——那会在窗口外，A*不知道外面长什么样。

所以需要选一个**在窗口内、有意义、能通向终点的中间点**。

#### 四级回退策略

系统设计了四个策略，按**优先级**逐个尝试：

**策略1：任务航点直连（最理想）**

条件：
1. 某个任务航点在窗口内
2. 从飞机当前位置到该航点的直线不碰障碍物

结果：直接取这个任务航点作为子目标

**为什么这是最好的？** 任务航点是用户真正想让你去的地方。如果能直接飞过去，不需要绕路。

**策略2：全局参考路径采样（次理想）**

条件：策略1失败了

做法：
1. 看全局参考路径（Informed RRT*算的那条粗路线）
2. 找出在窗口内的那一段
3. 在那些点上做SDF校核，挑安全的
4. 选离终点最近的安全点

**策略3：方向采样（保底策略）**

条件：策略1和2都失败了

做法：
1. 从飞机当前位置，朝着终点的方向
2. 在窗口边界上采样多个候选点
3. 做SDF校核，挑安全的
4. 评分选最好的

评分函数：

$$S(q) = (\|g-p\| - \|g-q\|) + 0.35 \cdot \text{clip}(d_{\text{sdf}}(q), 0, H) - 0.15 \cdot d(q, P_{\text{global}}) - 0.05 \cdot \|q-p\|$$

| 项 | 系数 | 含义 |
|----|------|------|
| $\|g-p\| - \|g-q\|$ | 1.0 | **终点接近度**：选了q之后，离终点近了多少 |
| $\text{clip}(d_{\text{sdf}}, 0, H)$ | 0.35 | **安全性**：q离障碍物多远 |
| $d(q, P_{\text{global}})$ | -0.15 | **偏离惩罚**：q离全局路径多远 |
| $\|q-p\|$ | -0.05 | **距离惩罚**：别太绕远 |

**策略4：BFS前沿探索（最后一招）**

条件：前三个策略都没找到好候选点

做法：
1. 从飞机当前位置开始，做BFS（广度优先搜索）
2. 像洪水一样往四周扩散
3. 找到自由空间的前沿点（再往前就是未知区域了）
4. 取这些前沿点作为候选

#### 子目标选择的流程图

```
开始选子目标
    │
    ▼
┌─────────────────────────┐
│ 策略1：窗口内有可达任务航点？ │
│ 是 → 取任务航点作为子目标    │
│ 否 → 继续                   │
└─────────────────────────┘
    │
    ▼
┌─────────────────────────────┐
│ 策略2：全局路径在窗口内有安全点？ │
│ 是 → 取离终点最近的安全点      │
│ 否 → 继续                     │
└─────────────────────────────┘
    │
    ▼
┌─────────────────────────────┐
│ 策略3：沿终点方向采样窗口边界    │
│ 是 → 评分选最好的候选点        │
│ 否 → 继续                     │
└─────────────────────────────┘
    │
    ▼
┌─────────────────────────────┐
│ 策略4：BFS洪水填充找自由前沿    │
│ → 取前沿点作为子目标           │
└─────────────────────────────┘
    │
    ▼
子目标确定 → 窗口A*规划局部路径
```

**答辩话术**："子目标选择设计四级回退策略，优先级从高到低为：任务航点直连、全局参考路径采样、方向边界采样、BFS自由前沿探索。评分函数综合终点接近度、障碍物距离、全局路径偏离度和路径长度四项指标，权重分别为1.0、0.35、-0.15和-0.05，在可达性、安全性、方向一致性和效率之间取得平衡。"

### 12.6 路径拼接（不能断链）

#### 为什么要拼接？

局部A*只规划到**子目标**（在窗口内），但子目标不是真正的终点。如果只飞到子目标就停下，飞机就"迷路"了。

#### 拼接公式

$$P_{\text{exec}} = P_{\text{local}} \oplus P_{\text{tail}}$$

| 符号 | 含义 |
|------|------|
| $P_{\text{local}}$ | 窗口A*算的局部路径（当前位置 → 子目标） |
| $\oplus$ | 路径拼接（去掉重复点，连起来） |
| $P_{\text{tail}}$ | 全局路径的剩余段（子目标 → 终点） |

```
局部路径：飞机 ──→ 子目标
                    ↓
全局剩余：         子目标 ──→ ... ──→ 终点
                    ↓ 拼接
执行路径：飞机 ──→ 子目标 ──→ ... ──→ 终点
```

**关键**：飞机每一步都有**完整到终点的路径**，不会出现"局部走完了不知道接下来去哪"。

#### 任务航点语义保护

如果子目标恰好是某个**任务航点**，拼接后要保证任务航点不被覆盖，飞到后标记"已完成"，继续往下个任务航点飞。

**答辩话术**："路径拼接机制将局部规划路径与全局路径尾段连接，保证每一步都有完整到终点的执行路径，避免局部路径完成后断链。任务航点语义独立保护，局部规划不覆盖任务完成状态，确保多航点任务的顺序执行。"

### 12.7 弧长前视跟踪（不盯着一个点）

#### 逐点跟踪的问题

如果路径上有很多点，飞机一个个追：

```
路径点：p1 → p2 → p3 → p4 → p5
飞机位置：     ✈

飞机盯着p2飞 → 快到p2了 → 转去盯p3 → 快到了 → 转去p4...
```

飞机在p2和p3之间会**急转弯**，轨迹像锯齿一样抖。

#### 前视跟踪的原理

不盯眼前的点，而是**沿着路径向前看一段距离**：

```
路径：p1 ── p2 ── p3 ── p4 ── p5 ── p6
              ✈（飞机在p2附近）
              │
              │  找到路径上离飞机最近的点 = p2
              │  沿路径向前走弧长 Ld = 5米
              │  到达 p4
              ▼
              ● ← 前视点（p4）

飞机朝着p4飞，而不是p2或p3
```

#### 安全校核

飞机当前位置 → 前视点的连线必须安全：

```
飞机 ──────────● 前视点
      ↑
   每隔一段距离采样一个点
   检查每个点的SDF > 安全阈值

   如果有任何一点不安全：
   → 放弃这个前视点
   → 重新选子目标
   → 重新规划
```

**答辩话术**："执行层采用弧长前视跟踪替代逐点跟踪：在路径上沿弧长前进前视距离确定参考点，避免密集路径点引起的执行振荡。同时执行飞机到前视点线段的等间隔SDF采样校核，若发现入侵则触发重规划，保证跟踪过程中的几何安全。"

### 12.8 速度前馈（提前告诉控制器）

#### 什么是前馈？

控制器通常根据**误差**来算控制量（反馈控制）：误差 = 目标位置 - 当前位置。

但这样有个**滞后**：等发现偏差了才修正。

**前馈**是提前告诉控制器"接下来该怎么动"：
- 路径是弯的 → 提前告诉飞机"要转弯了"
- 路径是直的 → 告诉飞机"保持当前速度"

#### 切向速度前馈

计算路径在当前点的**切线方向**和应该的速度大小：

$$v_{\text{ff}} = v_{\text{desired}} \cdot \frac{P'(s)}{\|P'(s)\|}$$

- $P'(s)$：路径在弧长s处的切向量
- $v_{\text{desired}}$：期望速度（弯的地方慢，直的地方快）

这个 $v_{\text{ff}}$ 直接送给控制器，和反馈控制量**叠加**。

#### 为什么在线模式更需要前馈？

在线规划中，路径可能频繁变化。如果没有前馈，控制器只根据位置误差反应，会**滞后**。加了前馈，控制器能**提前准备**，跟踪更平滑。

**答辩话术**："在线模式引入路径切向速度前馈：根据路径局部曲率计算期望切向速度，直接叠加到控制器输出。前馈项提供 anticipatory 控制分量，补偿纯反馈控制的相位滞后，改善动态路径跟踪的响应速度与平滑性。"

### 12.9 完整实时规划循环

```

  3. 初始化滑动窗口A*

每步循环（每0.01秒）：
  ├─ Step 1: 飞机当前位置 p_current
  │
  ├─ Step 2: 检查是否需要重规划
```

## 13. 编队切换方法

### 13.1 为什么要编队切换？

5个人并排走路：
- 排成**横排**（一字）：占路宽，看得广，适合搜索
- 排成**竖列**（纵队）：省空间，适合过窄道
- 排成**V字**（大雁）：空气动力学省油，互相掩护
- 排成**菱形**（钻石）：中心有人指挥，四周警戒

无人机编队也一样。不同任务需要不同队形。

**编队切换的核心问题**：

| 问题 | 如果做不好会怎样 |
|------|---------------|
| **撞机** | 变形时两架飞机轨迹交叉，直接撞上 |
| **丢通信** | 变完后某架飞机离其他人太远，信号断了 |
| **抖动** | 目标点跳来跳去，飞机急转弯来回晃 |
| **跟不上** | 领队转弯，从机惯性大，掉队了 |
| **故障了怎么办** | 一架飞机坏了，剩下4架怎么重组？ |

### 13.2 Leader-Follower架构

```
      从机2
        ↑
从机1 ← L(领队) → 从机3
        ↓
      从机4
```

**关键设定**：
- 只有**领队**做路径规划
- **从机**不规划路径，只算"我应该在领队的哪个相对位置"
- 所有从机的目标 = 领队位置 + 队形偏移

### 13.3 偏移量的数学表达

$$p_i^d = p_L + R_z(\psi_L) \cdot o_i$$

| 符号 | 含义 | 例子 |
|------|------|------|
| $p_i^d$ | 从机i的目标位置 | (12, 25, 5) |
| $p_L$ | 领队的当前位置 | (10, 20, 5) |
| $\psi_L$ | 领队的偏航角 | 30° |
| $o_i$ | 从机i在队形里的"座位号" | (2, 3, 0) |
| $R_z(\psi_L)$ | 绕z轴的旋转矩阵 | 把座位号转到领队的朝向 |

**为什么需要 $R_z(\psi_L)$？**

假设队形定义是"从机3在领队右边3米"。如果领队**左转90°**，那"右边"就不是原来的方向了——要跟着领队一起转。旋转矩阵 $R_z$ 就是做这件事。

### 13.4 切换触发条件

什么情况会触发编队切换？

| 场景 | 触发原因 | 例子 |
|------|---------|------|
| **任务需求** | 用户手动切换 | "进入峡谷，变纵队" |
| **障碍物约束** | 窄通道检测 | 过道太窄，横排过不去 |
| **故障重构** | 有飞机损坏 | 5架变4架，重新排队 |
| **动态调整** | 连通度下降 | 某从机偏离太远，队形收紧 |

### 13.5 平滑切换的核心方法：线性插值

**如果瞬间切换会怎样？**

假设从机原来目标在(10, 20)，下一秒突然变成(30, 40)。飞机为了追这个点，会**拼命加速**，可能超速、超倾角，甚至失控。

**解决办法：过渡动画**

$$o_i(t) = (1 - \alpha(t)) \cdot o_i^{\text{old}} + \alpha(t) \cdot o_i^{\text{new}}$$

$\alpha(t)$ 是一个**从0慢慢变到1**的函数：

```
t=0:   α=0,  目标 = 100%旧队形 + 0%新队形
t=1s:  α=0.3, 目标 = 70%旧 + 30%新
t=2s:  α=0.7, 目标 = 30%旧 + 70%新
t=3s:  α=1,  目标 = 0%旧 + 100%新  ← 切换完成
```

**$\alpha(t)$ 怎么选？**

| 插值方式 | 特点 | 公式 |
|---------|------|------|
| 线性 | 匀速变化 | $\alpha = t/T$ |
| Sigmoid | 先慢后快再慢 | $\alpha = 1/(1+e^{-k(t-T/2)})$ |
| 余弦 | 平滑加减速 | $\alpha = 0.5(1-\cos(\pi t/T))$ |

### 13.6 切换时的速度同步

编队切换不能只看位置，还要看**速度匹配**。

领队速度 = $v_L$，从机想要的速度 = $v_L + \dot{R}_z o_i$

因为队形在旋转时，相对位置的变化会产生**科氏速度**。从机控制器要预估这个附加速度，否则会有滞后。

**答辩话术**："编队切换采用线性插值实现平滑过渡，切换过程中从机目标位置为旧队形与新队形的凸组合，过渡系数 $\alpha(t)$ 随时间从0单调递增到1。插值函数可选斜坡、Sigmoid或余弦形式，避免目标跳变引起的控制器饱和与执行器过载。切换过程中从机目标速度不仅包含领队平移速度，还包含由队形旋转引起的附加速度项 $\dot{R}_z o_i$，减少队形转动时的从机跟踪滞后。"

### 13.7 拓扑连通度评估（图论选队形）

编队不仅要看形状，还要保证**飞机之间能通信**。

**通信图**：节点 = 飞机，边 = 能互相通信。

**邻接矩阵 $A$**、**度矩阵 $D$**、**拉普拉斯矩阵 $L = D - A$**。

**代数连通度 $\lambda_2$**（第二小的特征值）：
- $\lambda_2 > 0$：图连通，所有人能间接联系
- $\lambda_2$ 越大：通信网络越健壮

**论文 [4] Lan et al. (IEEE TAES 2025)**：故障后拓扑重构时，优先选择 $\lambda_2$ 最大的队形。

**答辩话术**："编队通信拓扑采用无向图建模，通过拉普拉斯矩阵 $L=D-A$ 的代数连通度 $\lambda_2$ 评估队形鲁棒性。$\lambda_2>0$ 当且仅当通信图连通，值越大代表拓扑对链路失效的容忍度越高。拓扑重构时以 $\lambda_2$ 作为队形优选指标，保证切换后的编队具有最强的通信冗余性。"

### 13.8 故障后的拓扑重构（5步流程）

```
Step 1: 检测到故障
        ↓
Step 2: 确定健康飞机数 N_h（比如原来5架，坏了1架，N_h=4）
        ↓
Step 3: 按 N_h 生成候选队形
        - V形4机版、菱形4机版、线形4机版...
        ↓
Step 4: 对每个候选队形，算通信图和 λ₂
        - 建邻接矩阵
        - 算拉普拉斯矩阵 L
        - 算 λ₂
        ↓
Step 5: 选 λ₂ 最大的队形，执行平滑切换
```

**论文 [4] 的核心贡献**：Lan 等人提出**最小能量约束下的拓扑重构**。本系统借鉴其"按健康飞机数重新生成队形并选连通性最强者"的思想。

### 13.9 候选队形生成规则

| $N_h$ | 候选队形 |
|-------|---------|
| 2 | 线形（前后） |
| 3 | 三角形、V形 |
| 4 | 菱形、V形、线形 |
| 5 | V形、菱形、三角形、线形 |

**关键**：不是简单"删掉坏飞机、其他人不动"。例如原来5架V形，中间领队+左右各两架。如果左翼最外侧坏了，直接删掉那个位置队形就偏了。系统会**重新生成对称的4架队形**（比如菱形或紧凑V形）。

**答辩话术**："拓扑重构不是简单地删除故障节点保留其余偏移量，而是按健康无人机数 $N_h$ 从队形库中重新生成对称候选队形。例如五机V形故障后，不是保留不对称四机分布，而是切换至四机菱形或紧凑V形，保证编队几何对称性与重心稳定性。"

### 13.10 编队切换时的避碰保障

#### 从机安全目标投影

编队过窄通道时，如果按标准队形，某架从机可能撞墙：

```
窄通道：
┌─────────────────┐
│                 │
│    L  从机3     │ ← 从机3卡在墙里！
│   ╱             │
│  F2             │
│                 │
└─────────────────┘
```

解决办法：把从机位置沿**到领队的方向**往回收缩：

$$p_i(\lambda) = p_L + \lambda \cdot \Delta_i$$

找最大的 $\lambda$ 使得：

$$\text{SDF}(p_i(\lambda)) \ge r_{\text{follower}}$$

**能维持多少队形就维持多少**，但不能撞墙。

**答辩话术**："窄通道场景执行从机安全目标投影：沿领队-从机偏移向量线性缩放，收缩系数 $\lambda^*$ 由SDF安全约束确定。投影在保持队形语义与保证碰撞安全之间做 trade-off，从机最大收缩至领队位置，避免编队穿越过程中因刚性队形导致的碰撞。"

#### 间距约束与APF排斥

即使队形正确，切换过程中两架飞机轨迹可能交叉。系统用**APF斥力**作为额外保险：

$$F_{\text{rep},ij} = k_{\text{rep}} \left(\frac{1}{d_{ij}} - \frac{1}{r_{\text{sep}}}\right)^2 \nabla d_{ij}$$

当飞机i和j的距离 $d_{ij}$ 小于安全间距 $r_{\text{sep}}$ 时，互相排斥。

**答辩话术**："编队切换过程中，除了队形几何约束外，还引入人工势场 inter-agent 排斥力。当任意两机距离低于安全分离半径时激活斥力项，作为队形约束之外的动态碰撞保险，防止过渡轨迹交叉引起的碰撞风险。"

### 13.11 完整切换流程（代码层面）

```
每帧循环：
├─ 1. 检查是否触发切换
│   ├─ 用户指令？ → 切到指定队形
│   ├─ 故障检测报警？ → 启动拓扑重构
│   └─ 窄通道检测？ → 临时收缩或切线形
│
├─ 2. 若触发切换：
│   ├─ 旧队形 = current_formation
│   ├─ 新队形 = target_formation
│   └─ 初始化过渡计时器 t = 0
│
├─ 3. 过渡中（0 < t < T_transition）：
│   ├─ α = t / T_transition
│   ├─ 对每个从机 i：
│   │   o_i(t) = (1-α)·o_i^old + α·o_i^new
│   │   p_i^d = p_L + R_z(ψ_L)·o_i(t)
│   │   v_i^d = v_L + d/dt[R_z(ψ_L)·o_i(t)]
│   └─ t += dt
│
├─ 4. 切换完成后：
│   ├─ current_formation = target_formation
│   └─ 过渡计时器关闭
│
├─ 5. 正常跟踪（每帧都执行）：
│   ├─ 从机目标 = p_L + R_z(ψ_L)·o_i
│   ├─ 窄通道？→ 执行安全投影
│   ├─ 间距太近？→ 叠加APF排斥
│   └─ 送入控制器跟踪
│
└─ 6. 算连通度（监控）：
    ├─ 建邻接矩阵 A
    ├─ 算 Laplacian L = D - A
    ├─ 算 λ₂
    └─ λ₂ < 阈值？→ 报警或触发重构
```

## 15. 答辩Q&A预备

### Q1：为什么用串级PID而不是更先进的控制？

**A**：串级PID是工业界最成熟、参数最易懂的四旋翼控制方案。反步法和滑模在理论上更漂亮，但PID的工程调参经验最丰富，仿真复现最稳定。三种控制器我都在代码里实现了，可以对比。

### Q2：RRT*和A*怎么选？

**A**：A*在栅格上最优但受分辨率限制；RRT*在连续空间渐近最优，适合复杂几何。A*算得快适合在线，RRT*适合离线粗规划。本系统把它们放在不同层级各司其职。

### Q3：GNN和深度学习有什么关系？

**A**：本系统的GNN是神经动力学活动扩散模型，源于Grossberg的生物神经元方程，**没有训练过程**。它利用并行扩散特性快速收敛，和深度学习的图神经网络（如GCN、GAT）是完全不同的数学机制。

### Q4：Hybrid A*在四旋翼上是否必要？

**A**：四旋翼本身是小机动飞行器，理论上不需要Hybrid A*的运动学约束。但本系统实现Hybrid A*是为了展示航向连续性，并验证其在非完整约束载体上的通用性。在格子间等需要精确姿态控制的场景有实用价值。

### Q5：在线重规划的窗口多大？怎么选？

**A**：窗口大小20米，是计算量与前瞻性的折中。太小前瞻不足容易进死胡同，太大局部A*计算慢达不到实时。通过任务航点可见性、全局路径采样、方向采样和BFS前沿四级策略选择子目标，保证窗口内总有有效目标。

### Q6：八层安全链路是不是过度设计？

**A**：各层针对不同时间尺度和失效模式。全局规划失效时用局部重规划，局部失效用APF，APF失效用SDF屏障。这是"纵深防御"思想，航天、核电等安全关键系统普遍采用。

### Q7：为什么不用分布式编队，而用Leader-Follower？

**A**：Leader-Follower结构简单、计算量集中在领队，从机只需跟踪相对位置，适合实时仿真。分布式需要每架都跑完整规划+协商一致，通信量和计算量更大。本系统重点在规划算法验证，Leader-Follower降低了编队本身的复杂度，让重点更突出。

### Q8：故障后如果 $\lambda_2$ 最大的队形还是不够安全怎么办？

**A**：系统有两层保险：一是 $\lambda_2$ 阈值检查，若所有候选队形的 $\lambda_2$ 都低于安全阈值，则触发紧急解散（各机独立避障）；二是在执行层，无论队形如何，APF互斥力和SDF安全屏障始终激活，保证物理不碰撞。

### Q9：切换时间 $T_{\text{transition}}$ 怎么选？

**A**：与编队尺度和飞行速度相关。代码默认2-3秒，经验上使过渡期间最大加速度不超过控制器饱和限的60%。过短则飞机跟不上，过长则反应不及时。

### Q10：实时规划的路径质量比离线差多少？

**A**：理论上是次优的，因为只看了局部。但通过全局参考路径引导、ESDF软代价引导、以及捷径代价，实际路径长度与离线最优的差距通常在10%以内，而计算速度提升3个数量级。
