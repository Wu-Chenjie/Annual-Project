# CPP重构 使用说明

## 1. 这份文档解决什么问题

1. 如何编译和运行 C++ 版仿真。
2. 如何通过 `config.hpp` 切换预设场景和调整参数。
3. 当前支持哪些功能，与 Python 版的对应关系。

---

## 2. 快速开始

### 第一步：编译

```bash
cd CPP重构
g++ -std=c++20 -O3 -Iinclude \
  src/rotor.cpp src/allocator.cpp src/wind_field.cpp src/smc.cpp \
  src/controller.cpp src/topology.cpp src/drone.cpp \
  src/formation_simulation.cpp src/visualization.cpp \
  src/occupancy_grid.cpp src/map_loader.cpp \
  src/artificial_potential_field.cpp src/obstacle_scenario.cpp \
  src/main.cpp -o build/sim_main
```

### 第二步：运行

```bash
./build/sim_main
```

### 第三步：查看结果

输出在 `outputs/`：
- `trajectory_3d.png` — 三维轨迹图
- `error_realtime_3d.png` — 实时误差图
- `error_statistics.png` — 误差统计图

## 3. 使用 config.hpp 切换场景

### 3.1 一行切换预设

```cpp
#include "config.hpp"

int main() {
    using sim::get_config;
    using sim::FormationSimulation;
    using sim::ObstacleScenarioSimulation;

    // 选一个预设：
    auto config = get_config("basic");           // 基础编队
    auto config = get_config("obstacle");         // 简单避障
    auto config = get_config("warehouse");         // 仓库 Hybrid A*
    auto config = get_config("warehouse_a");       // 仓库 A*
    auto config = get_config("warehouse_online");  // 仓库在线
    auto config = get_config("custom");            // 自定义

    if (config.enable_obstacles) {
        ObstacleScenarioSimulation sim(config);
        // 如果用硬编码障碍物替代 JSON，加一行：
        // sim.set_obstacles(make_warehouse(), {Vec3{-3,-3,0}, Vec3{45,28,10}});
        auto result = sim.run();
    } else {
        FormationSimulation sim(config);
        auto result = sim.run();
    }
}
```

### 3.2 预设场景说明

| 预设 | 从机 | 控制器 | 规划器 | 模式 | 地图 |
|------|------|--------|--------|------|------|
| `basic` | 3 | PID+SMC | — | — | 无 |
| `obstacle` | 2 | PID+SMC | A* | 离线 | 三柱 |
| `warehouse` | 3 | Backstep+SMC | Hybrid A* | 离线 | 仓库 |
| `warehouse_a` | 3 | Backstep+SMC | A* | 离线 | 仓库 |
| `warehouse_online` | 3 | Backstep+SMC | A* | 在线 | 仓库 |

### 3.3 自定义配置

编辑 `config.hpp` 中的 `config_custom()` 函数：

```cpp
inline ObstacleConfig config_custom() {
    ObstacleConfig c;
    c.max_sim_time = 65.0;          // 仿真时长
    c.use_backstepping = true;      // 反步法位置环
    c.num_followers = 3;            // 从机数
    c.formation_spacing = 0.5;      // 编队间距
    c.planner_kind = "hybrid_astar"; // 规划器
    c.planner_mode = "offline";     // 离线/在线
    // ... 更多参数见下方
    return c;
}
```

## 4. 参数速查

### 编队

| 字段 | 类型 | 说明 |
|------|------|------|
| `num_followers` | int | 从机数量 |

| `initial_formation` | string | 初始队形："v_shape"/"diamond"/"line"/"triangle" |
| `formation_schedule` | vector | 切换计划：`{{时间, 队形, 过渡时长}, ...}` |

### 控制器

| 字段 | 类型 | 说明 |
|------|------|------|
| `use_smc` | bool | SMC 姿态环 |
| `use_backstepping` | bool | 反步法位置环（替代 PID） |
| `leader_gain_scale` | double | 领航机增益缩放 |
| `follower_gain_scale` | double | 从机增益缩放 |
| `leader_max_vel` | double | 领航机最大速度 (m/s) |
| `leader_max_acc` | double | 领航机最大加速度 (m/s²) |

### 航点

| 字段 | 类型 | 说明 |
|------|------|------|
| `waypoints` | `vector<Vec3>` | 航点列表 |
| `wp_radius` | double | 航点到达半径 (m) |
| `wp_radius_final` | double | 终点到达半径 (m) |

### 障碍物与避碰

| 字段 | 类型 | 说明 |
|------|------|------|
| `enable_obstacles` | bool | 启用障碍物 |
| `map_file` | string | 地图 JSON 路径 |
| `safety_margin` | double | 安全裕度 (m) |
| `detect_margin_scale` | double | 检测阈值缩放 |
| `planner_sdf_aware` | bool | SDF 精判薄障碍物 |

### 路径规划

| 字段 | 类型 | 说明 |

| `planner_kind` | string | "astar" / "hybrid_astar" |

### 在线模式

| 字段 | 类型 | 说明 |
|------|------|------|

| `formation_recovery_clearance_margin` | double | 恢复判定额外安全裕度 (m) |

## 5. 常用操作

### 只跑基础编队

```cpp
auto config = get_config("basic");
FormationSimulation sim(config);
```

### 切换规划器

```cpp
auto config = get_config("custom");
config.planner_kind = "astar";        // 更快
config.planner_kind = "hybrid_astar"; // 更平滑
```

### 切换控制器

```cpp
auto config = get_config("custom");
config.use_backstepping = true;   // 反步法+SMC
config.use_backstepping = false;  // PID+SMC
```

### 缩小/放大编队

```cpp
auto config = get_config("custom");
config.formation_spacing = 0.3;   // 紧凑（窄通道）
config.formation_spacing = 1.0;   // 松散（开阔区）
```

### 切换在线模式

```cpp
auto config = get_config("custom");
config.planner_mode = "online";
config.sensor_enabled = true;
config.planner_replan_interval = 2.0;  // 每 2s 重规划
```

## 9. 常见问题

**Q: 编译报错？**
- 确认编译器支持 C++20（g++ 11+ 或 clang 14+）
- 确认在 `CPP重构/` 目录下执行编译命令

**Q: 地图加载失败？**
- `map_file` 路径相对于 `CPP重构/` 目录，应为 `"../maps/sample_warehouse.json"`
- 或用 `set_obstacles()` 硬编码障碍物

**Q: 碰撞太多？**
- 减小 `leader_max_vel`
- 切换 `planner_kind="hybrid_astar"`

**Q: C++ 结果与 Python 不一致？**
- C++ 浮点精度与 Python 有微小差异（`world_to_index` 使用 `std::floor`）
- SDFAwareGrid 和路径校核细节可能有差异
- 控制器参数已对齐，误差应在合理范围

## 10. 障碍场景结果口径

普通 C++ 障碍场景 `ObstacleScenarioSimulation::run()` 现在除基础误差指标外，还会在结果层维护：

- `safety_metrics.min_inter_drone_distance`
- `safety_metrics.downwash_hits`

在 `src/warehouse_main.cpp` 中会额外打印：

```text
Safety: min_inter=<value> downwash_hits=<value>
```

并额外写出：

```text
outputs/warehouse_result.json
```

其中当前稳定导出的字段包括：

- `task_waypoints`
- `replanned_waypoints`
- `executed_path`
- `fault_log`
- `safety_metrics`

仓库场景对应的自动化检查位于：

```text
tests/test_obstacle_scenario.py::test_cpp_warehouse_writes_obstacle_result_json
```

如需从该结果直接生成风险报告，可在项目根目录执行：

```powershell
python tools/cpp_risk_report.py cpp/outputs/warehouse_result.json --safety-profile indoor_demo
```

默认输出到：

```text
outputs/reports/warehouse_result_risk_report.json
outputs/reports/warehouse_result_risk_report.md
```

这两个指标只代表普通 C++ 障碍场景主流程中的 follower 位置统计，不应直接外推为 dynamic replay 的完整编队安全统计。dynamic replay 仍主要用于 leader 轨迹展示和部分安全语义验证。
